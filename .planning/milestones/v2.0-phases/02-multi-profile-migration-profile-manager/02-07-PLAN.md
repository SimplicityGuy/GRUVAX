Backend contract (Plan 02-05, read 02-05-SUMMARY for exact shapes):
  GET /api/admin/profiles → [{id, display_name, last_sync_at, last_sync_status, last_sync_item_count, app_token_revoked, status}]  (status ∈ connected|pending|syncing|re-auth-required)
  GET /api/admin/profiles/{id} → single profile (the D2-13 poll target)
  POST /api/admin/profiles {display_name}
  PATCH /api/admin/profiles/{id} {display_name}
  POST /api/admin/profiles/{id}/connect {pat}  → 200 connected | 401 pat_rejected | 409 user_id_collision | 503 upstream
  POST /api/admin/profiles/{id}/rotate {pat}   → 200 | 409 user_id_mismatch | 401 pat_rejected
  POST /api/admin/profiles/{id}/sync           → 202 accepted
  DELETE /api/admin/profiles/{id}              → 200 | 409 (default profile not deletable)
  All mutations need the X-CSRF-Token header (read gruvax_csrf cookie — adminClient.ts already does this).

Design tokens (the contract per CLAUDE.md "consume tokens; never hardcode hex"):
  --gruvax-success (#1A7A4A), --gruvax-warning (#E6A800), --gruvax-error (#C0392B), --gruvax-blue (#0051A2), --gruvax-blue-faint (#EEF5FB) are all defined in design/gruvax-design-tokens.css and wired into the frontend. The status-badge tints are derived state colors built FROM these tokens via color-mix — not new brand primitives, so no new token variables are added.

## Trust Boundaries

| admin browser → /api/admin/profiles/* | PIN-gated; the pasted PAT (secret) is entered in the drawer and sent in the request body. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07-01 | Information Disclosure | PAT visible/leaked in the UI | mitigate | PAT input type="password" + show/hide toggle; the instruction states the token is never shown again after connecting; the API never returns the stored PAT (Plan 02-05). |
| T-02-07-02 | Spoofing | CSRF on profile mutations | mitigate | All mutations send X-CSRF-Token via adminClient.ts (double-submit, carries from v1); require_admin enforces it server-side. |
| T-02-07-03 | Tampering | XSS via profile display_name in cards/drawer | mitigate | All strings via JSX interpolation, never innerHTML (UI-SPEC + project convention). |
| T-02-07-04 | Information Disclosure | raw HTTP error codes / stack detail shown to the owner | mitigate | Error type discriminators mapped to the UI-SPEC friendly plain-language copy; no status codes surfaced. |
| T-02-07-SC | Tampering | npm installs | mitigate | No new npm packages (lucide-react / TanStack Query / Zustand already locked). If `npm run build` pulls a new transitive package, STOP and verify via npmjs.com. |

No HIGH-severity unmitigated threats.
</threat_model>
