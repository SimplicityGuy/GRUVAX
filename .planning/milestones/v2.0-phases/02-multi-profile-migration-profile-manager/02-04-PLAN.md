## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LAN browser → /api/session, /api/session/bind | No-PIN browse-binding crosses here; binding is derived/validated server-side, never a global "see all" view. |
| browse-binding cookie ↔ admin PIN session | Two independent cookies; must not couple (D2-10). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Spoofing | forged gruvax_browse_binding cookie | mitigate | Server validates the bound profile_id against the active-profiles set / registry on every per-profile endpoint (D2-04, enforced in Plan 02-02/02-03 deps); a forged id resolving to no active profile → 404/403. Home-LAN scope (OOS-06: no TLS yet); Secure flag activates with future TLS. |
| T-02-04-02 | Spoofing | admin-session confused with browse-binding | mitigate | Distinct cookie names; require_admin reads only gruvax_session/gruvax_csrf; test_binding_independent_of_admin asserts no coupling (D2-10). |
| T-02-04-03 | Information Disclosure | GET /api/session leaking secrets (PAT) in profiles[] | mitigate | Bootstrap SELECT excludes app_token_encrypted/discogsography_user_id; returns only display_name + sync metadata. |
| T-02-04-04 | Tampering | CSRF on bind/unbind | accept | SameSite=Strict on both cookies + same-site-only LAN traffic blocks cross-site POST; bind/unbind are non-destructive (reversible via picker). Admin mutations keep the existing CSRF double-submit. |
| T-02-04-SC | Tampering | npm/pip installs | accept | No package installs (RESEARCH §Package Legitimacy Audit: none new). |

No HIGH-severity unmitigated threats.
