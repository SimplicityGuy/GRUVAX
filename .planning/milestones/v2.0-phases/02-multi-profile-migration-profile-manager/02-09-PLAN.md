<objective>
Close UAT Gap (test 5, minor): a freshly-created+synced profile has 0 cube boundaries, so a search result that IS in the collection lands no cube and the kiosk silently highlights nothing. The user can't tell whether search is broken or the shelf layout simply isn't configured for this profile.

Diagnosis (from 02-UAT.md): search scoping works and the locate→cube path is functional on a boundary-configured profile (verified on Default: `/api/locate?release_id=1351` → primary_cube + confidence 0.85). For a profile with zero `cube_boundaries`, `/api/locate` returns HTTP 200 with `primary_cube: null`, `label_span: []`, `confidence: 0.0` — the documented "release IS in collection but no cube boundary covers its label" case (locate.py lines 17-20). This is genuinely profile-scoped because locate uses the per-profile segment cache.

Scope of THIS plan: the kiosk affordance/feedback only. When a selected result resolves to a null-cube / zero-confidence locate (the record IS in the collection — HTTP 200, not 404), show a Nordic-Grid message telling the user the shelf layout for this collection isn't configured yet, instead of silently highlighting nothing.

OUT OF SCOPE (explicit deferral): the deeper onboarding path — an admin flow to configure cube boundaries for a newly-created profile — is larger and is deferred to a future phase. Phase 3 is "Devices + pairing"; the boundary-onboarding flow for new profiles should be captured for Phase 4 (Sync polish + diagnostics) or a dedicated boundary-onboarding phase. This plan does NOT implement boundary configuration; it only makes the missing-layout state legible to the user.

Purpose: Replace a silent dead-end with a clear, on-brand explanation so the owner knows the next step is to configure the shelf layout for this profile (not that search is broken).
Output: A `shelfLayoutUnavailable` derived flag in the store, a `ShelfLayoutNotConfigured` Nordic-Grid affordance component with a vitest test, and KioskView wiring that renders it when a locate result lands no cube.
</objective>
