<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SPA → /api/session, /api/events/{profile_id}, /api/search, /api/locate | The client reads the browse cookie to build per-profile URLs; the server re-validates every profile_id (D2-04) — the client is not trusted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06-01 | Spoofing | SPA sending a profile_id it isn't bound to | mitigate | Server-side per-profile deps 403 on mismatch (Plan 02-02/02-03); the SPA only ever uses the cookie-bound id, and a mismatch is silently redirected to /select per UI-SPEC error contract. |
| T-02-06-02 | Information Disclosure | picker exposing PAT/secret in profiles[] | mitigate | GET /api/session returns only display_name + sync metadata (enforced server-side, Plan 02-04); the SPA renders only those fields. |
| T-02-06-03 | Tampering | XSS via profile display_name in cards | mitigate | All strings rendered via JSX interpolation, never innerHTML (UI-SPEC + project convention). |
| T-02-06-SC | Tampering | npm installs | mitigate | No new npm packages (RESEARCH: lucide-react, TanStack Query, Zustand, React Router all already locked). If `npm run build` pulls a new transitive package, STOP and verify via npmjs.com before proceeding. |

No HIGH-severity unmitigated threats.
</threat_model>
