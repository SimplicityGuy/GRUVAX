<objective>
Fix the `just compose-smoke` blocker reported in 01-HUMAN-UAT.md Test 1. The `init-sync` service in `compose.yaml` was implemented (commit 4dc644a) with a redundant `build:` block that tags the SAME image as the `api` service (`ghcr.io/simplicityguy/gruvax:latest`). BuildKit's exporter rejects the duplicate-tag parallel export with `target api: failed to solve: image already exists`, killing compose-up before any container starts.

Plan 01-05 line 449 originally specified `image: ghcr.io/simplicityguy/gruvax:latest   # reuses main gruvax image` with NO `build:` block for init-sync. This plan restores that intent by deleting the redundant `build:` block. The `depends_on.api: { condition: service_healthy }` (lines 259-261) already guarantees the api service has built and tagged the image before init-sync starts, so init-sync can safely reuse the locally-tagged image without rebuilding.

Purpose: Unblocks UAT Test 1 and downstream tests 2-5 (currently skipped). Closes the only Phase 1 gap.

Output: A single-block deletion in `compose.yaml` plus a YAML comment documenting the intentional image reuse so a future contributor doesn't pattern-match the api service and re-introduce the duplicate build.
</objective>
