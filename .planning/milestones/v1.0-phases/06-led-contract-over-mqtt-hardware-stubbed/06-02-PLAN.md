<objective>
Deliver the highlight-lifecycle vertical slice: a configurable idle/ambient
baseline on every cube, a server-scheduled timed revert of active highlights back
to that baseline (TTL or next-search, whichever first), and an optional retain
mode that accumulates a recently-found trail with independent per-highlight
timeouts. This replaces the original "highlight lingers until 4h expiry" model with
an explicit, server-owned, fully-testable lifecycle.

Purpose: Closes LED-11 (idle/ambient baseline), LED-12 (timed server-scheduled
revert), and LED-13 (retain mode). After this slice, a kiosk select lights the
cube, the cube returns to the ambient baseline after the TTL (or when the next
search fires), and flipping retain mode on makes selections accumulate instead.

Output: A new src/gruvax/mqtt/lifecycle.py owning the cancelable revert registry,
an ambient-baseline publisher, and the lifecycle-aware illuminate path wired into
the public /api/illuminate endpoint and the app lifespan.
</objective>

<design_constraints>
- Server-scheduled, not firmware-dependent (D-22): the revert is an asyncio task in gruvax-api, NOT MQTT message-expiry. Expiry remains only the safety backstop on retained state/* (Plan 01); the lifecycle owns the real revert.
- Ambient is the persistent baseline (D-20): publish_ambient writes the retained state/* (qos=1, retain=True, expiry props) for a cube using led_color.ambient + led_brightness.ambient. Revert == publish_ambient for the reverted cubes.
- Registry must be bounded/leak-free (security): each revert task removes its own entry from the registry in a finally; cancel_and_revert_all is called on shutdown. Key entries by a highlight_id (uuid) → set of affected CubeRefs + the task handle.
- Injectable delay/clock for testability (D-22): schedule_revert takes a sleep callable / delay value so tests pass ~0 and assert the ambient republish synchronously; never time.sleep, never a real 180s wait in tests.
- Default vs retain branch (D-21/D-23): read led_highlight.retain_mode at illuminate time. Default → cancel+revert the single prior active highlight before lighting the new one, schedule a TTL revert keyed active_ttl_seconds. Retain → do NOT cancel prior; just add the new highlight and schedule its independent revert keyed retain_ttl_seconds.
- Degraded mode (client=None): every lifecycle function short-circuits with a logged warning and never raises (mirror Plan 01 / mqtt/client.py posture).
</design_constraints>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| kiosk browser → POST /api/illuminate | Untrusted client body triggers a server-scheduled lifecycle (public, unauthenticated per D-03) |
| in-process revert registry (app.state) | Server-owned asyncio tasks scheduling ambient reverts; must not grow unbounded |
| gruvax-api → Mosquitto broker | Internal Compose network only; no host port (DEP-03) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-18 | DoS | Revert-task registry unbounded growth | mitigate | Each revert task pops its own registry entry in a finally; default mode cancels the prior entry; cancel_and_revert_all empties the registry on shutdown — registry size stays O(active highlights) |
| T-06-19 | DoS | Repeated rapid illuminate calls spawning unbounded tasks | mitigate | Default mode cancels the prior task before scheduling a new one (one active highlight); retain mode is bounded by retain_ttl_seconds expiry of each task |
| T-06-20 | DoS | Broker hiccup during a revert blocking the loop | mitigate | Reverts use safe_publish native timeout + asyncio.gather(return_exceptions=True); degraded mode (client=None) short-circuits |
| T-06-21 | Tampering | TTL/retain values forced out of range via settings | mitigate | TTL/retain values read as ints with .get defaults; settings writes are admin-gated + range-validated in Plan 06-03; non-numeric falls back to the default |
| T-06-22 | Denial of teardown | Pending tasks surviving shutdown | mitigate | cancel_and_revert_all in lifespan teardown cancels every task in a try/except |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages this phase (RESEARCH Package Legitimacy Audit: zero new deps) |
</threat_model>
