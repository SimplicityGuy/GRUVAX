# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: Segment-Aware Position Precision.
> Answers: "What visual and interaction contracts does the admin cut-point + width-override
> editor need?" Generated by gsd-ui-researcher from locked sketch findings and upstream
> context artifacts (D-01..D-09). All design decisions are pre-populated; none were
> re-opened or re-decided here.

**Scope of this document:** Phase 5 introduces a new admin editor surface — the
**segment strip editor** (per-bin proportional view with drag-to-override handles) and
the **cut-point editor** (bin-card list with insert-cut + slide-up record picker). The
new surface **replaces** the Phase 3 per-cube first/last editor for all boundary editing.
The kiosk's sub-cube bar (Phase 2) is unchanged visually; this phase improves its
accuracy through the backend estimator, not through a UI change. The only
user-visible kiosk change is that the SubCubeBar now reflects two-level interpolation
estimates — the bar component contract is unchanged.

## Spacing Scale

Source: `design/gruvax-design-tokens.css` — `--gruvax-space-*` scale. All values
are multiples of 4px. Use token names in CSS; never write raw pixel values.

| Token               | Value  | Usage in this phase                                                                  |
|---------------------|--------|--------------------------------------------------------------------------------------|
| `--gruvax-space-1`  | 4px    | Strip drag-handle inner grip gap; icon-to-text gaps; A–Z rail item spacing         |
| `--gruvax-space-2`  | 8px    | Bin-card list-item internal padding (compact areas); chip row gap                  |
| `--gruvax-space-3`  | 12px   | Legend row vertical padding; straddle caption gap below strip                      |
| `--gruvax-space-4`  | 16px   | Card horizontal padding; bin-card section gap; sheet header inset                  |
| `--gruvax-space-5`  | 24px   | Card-to-card gap in the cut-point list; major intra-card section gap               |
| `--gruvax-space-6`  | 32px   | Between major layout sections (locator header → editor body)                       |
| `--gruvax-space-7`  | 48px   | Bottom-sheet slide-up clearance above safe-area                                    |
| `--gruvax-space-8`  | 64px   | Not used in this phase (mobile viewport constraint)                                |

**Phase 5 exceptions:**

- **Drag handles:** Touch target around each handle is minimum 44×44pt (WCAG 2.5.5). The
  visible grip pill is 6×46px (small by design — reads as a "gap between segments"), but the
  invisible pointer-event area extends to 44px wide on each side. Use `padding` + negative
  margins to achieve this without affecting strip layout.
- **Bin-card square chip:** 40×40px — matches the admin compact cell size
  (`--gruvax-cell-size-md`) so the locator header and bin-number chips read as the same
  grid vocabulary.
- **Insert-cut divider tap target:** Minimum 44px tall, full card width.
- **Record-picker rows:** Minimum 44px tall (touch floor), full sheet width.

### Segment fill color assignment

Segment fills cycle through the blue family by index within each bin. Use this
ordered rotation so adjacent segments are visually distinct:

| Segment index (within bin) | Fill token          | Text on fill          |
|----------------------------|---------------------|-----------------------|
| 0                          | `--gruvax-blue`     | `--gruvax-white`      |
| 1                          | `--gruvax-blue-light` | `--gruvax-blue-dark` |
| 2                          | `--gruvax-blue-dark` | `--gruvax-white`     |
| 3+                         | cycle from index 0  | same rule             |

The cycle resets per bin — bin 2's first segment is always index 0. This ensures
each bin reads as its own complete story, not a continuation of the previous bin's
palette sequence.

### Accessibility constraints (from design-language.md)

- Blue on white: 7.2:1 (AAA) — safe for all body text in legend rows and chips.
- Yellow-on-blue / blue-on-yellow: 3.1:1 — large text (18px+) or decoration only.
  The segment name overlay on blue fills uses white text, not yellow. Yellow is
  only used as the 5px override accent bar at the segment top, not as text color
  on colored fills. The "NEW" badge is yellow bg + `--gruvax-blue-dark` text (≥18px
  or icon-only context) — use Barlow Condensed 700 11px for the badge at this size;
  apply blue text to meet contrast floor.
- All override/auto detail text lives on **white legend rows** (`--gruvax-white` or
  `--gruvax-off-white` backgrounds), never on a colored segment fill.

### B. Segment Strip (`SegmentStrip`)

**File:** `frontend/src/routes/admin/SegmentStrip.tsx`

**Purpose:** Horizontal proportional bar representing one bin's segments. Widths are
fractions (always sum 100%). Used in both the per-bin editor (full-size) and the
bin-card cut list (compact mini variant).

**Props contract:**
```typescript
interface Segment {
  label: string            // Label name (ALL CAPS display)
  fraction: number         // 0–1; sum of all segments in bin === 1.0
  isOverride: boolean      // true if fraction was set by admin
  autoFraction: number     // count-derived fraction (always present)
  continues: boolean       // true if this label runs into the next bin (straddle right edge)
}

interface SegmentStripProps {
  segments: Segment[]
  onDragSetOverride?: (index: number, newFraction: number) => void  // undefined = read-only mini
  isReadOnly?: boolean     // true for bin-card mini strips
}
```

**Visual spec — full-size strip:**

| Property             | Value                                                                               |
|----------------------|-------------------------------------------------------------------------------------|
| Height               | 88px                                                                                |
| Border               | `1.5px solid var(--gruvax-blue)`                                                   |
| Border radius        | `var(--gruvax-radius-md)` (8px)                                                    |
| Overflow             | `hidden`                                                                            |
| Segment padding      | `8px 10px` (internal to each segment cell)                                         |
| Segment name text    | Barlow Condensed 700 16px ALL CAPS; white on dark fills, `--gruvax-blue-dark` on light fills |
| Segment name drop    | When segment width < 16% of total strip width: remove name from strip; keep only in legend |
| Override accent      | 5px top bar: `background: var(--gruvax-yellow); box-shadow: var(--gruvax-shadow-led)` — displayed on any segment where `isOverride === true` |
| Straddle treatment   | Segments where `continues === true` get `mask-image: linear-gradient(90deg, #000 calc(100% - 14px), transparent)` on the right edge |
| Width transition     | `transition: width var(--gruvax-duration-fast) linear` — smooth as drag adjusts   |

**Visual spec — mini strip (read-only, in bin-card):**

| Property             | Value                                                         |
|----------------------|---------------------------------------------------------------|
| Height               | 24px                                                          |
| Border               | `1px solid var(--gruvax-border-light)`                       |
| Border radius        | `var(--gruvax-radius-sm)` (4px)                              |
| Overflow             | `hidden`                                                      |
| Segment names        | Not shown (too small)                                         |
| Override accent      | 3px top bar (same yellow token, no glow — glow is too heavy at this scale) |
| Straddle treatment   | Same fade mask, but only 8px fade (shorter at this scale)    |

**Drag handle spec (full-size strip only):**

| State           | Grip appearance                                                                                 |
|-----------------|-------------------------------------------------------------------------------------------------|
| Passive         | `width: 6px; height: 46px; border-radius: var(--gruvax-radius-pill); background: var(--gruvax-white); border: 1.5px solid var(--gruvax-blue)` |
| Hover / active  | `background: var(--gruvax-yellow); border-color: var(--gruvax-yellow-dark); box-shadow: var(--gruvax-shadow-led)` |
| Touch target    | 44px wide (invisible expansion via padding/margin); 88px tall (full strip height)             |
| Cursor          | `col-resize`                                                                                    |
| ARIA            | `role="slider"`, `aria-valuemin`, `aria-valuemax`, `aria-valuenow` (percentage, integer)      |

**Drag interaction contract (pointer-events, touch-safe — no innerHTML):**

```js
// Pointer capture pattern — conserve sum between adjacent pair only
handle.addEventListener('pointerdown', ev => {
  handle.setPointerCapture(ev.pointerId)
  const rect = strip.getBoundingClientRect()
  const left  = sumFractions(segs, 0, i)      // sum of segs[0..i-1]
  const right = sumFractions(segs, 0, i + 2)  // sum of segs[0..i]
  const MIN   = 0.05                           // 5% minimum per segment

  const move = ev => {
    let pos = ((ev.clientX - rect.left) / rect.width)
    pos = Math.max(left + MIN, Math.min(right - MIN, pos))
    segs[i].fraction   = pos - left
    segs[i+1].fraction = right - pos   // sum preserved
    renderAll()
  }
  const up = () => {
    handle.removeEventListener('pointermove', move)
    handle.removeEventListener('pointerup', up)
    onDragSetOverride(i, segs[i].fraction)  // marks both adjacent segs as overridden
  }
  handle.addEventListener('pointermove', move)
  handle.addEventListener('pointerup', up)
})
```

Build strip DOM with `el(tag, props, ...kids)` helper + `node.replaceChildren()`.
Never use `innerHTML`.

when `abs(overrideFraction - autoFraction) > 0.03` (3 percentage points). Below this
threshold, the OVERRIDE chip shows without the caution icon. This matches the precision
the admin can meaningfully control via drag (drag resolution is ~1% at 390px strip width).

## Interaction Contracts

### State machine: segment strip drag

```
[strip at rest] → pointerdown on handle → [dragging: handle active yellow]
  → pointermove: redistribute segs[i] + segs[i+1] only; sum preserved; clamp [left+5%, right-5%]
  → pointerup: setPointerCapture released → [override staged: both adjacent segs marked as overridden]
  → segs re-render; PREVIEW CHANGES button enables
```

- Drag feedback: width transitions at `var(--gruvax-duration-fast)` 150ms.
- Snap: no snap — continuous drag. Percentages shown in legend update live.
- Min segment: 5% (`MIN = 0.05`). Clamped in the drag handler; handle cannot push a
  segment below this threshold.
- After drag, both adjacent segments are marked `isOverride: true` (dragging implies
  intent on both sides of the boundary).

### State machine: insert cut

```
[cut-point list] → tap "＋ insert cut" divider → [RecordPickerSheet opens in insert mode]
  → step 1: label pick (autocomplete / A–Z rail)
  → step 2: catalog# pick (DM Mono, phantom-validated)
  → tap "INSERT CUT":
      → phantom check passes: new bin card appears with "NEW" badge; renumber hint shown;
        all subsequent bin display numbers +1; PREVIEW CHANGES enables
      → phantom blocks: validation error; sheet stays open
  → tap "CANCEL" / swipe down: sheet dismissed; no change
```

### State machine: edit cut point

```
[bin card] → tap "✎ EDIT SEGMENTS" → SegmentEditorPanel expands inline (or sub-route)
  → within panel: tap on "starts at" row to edit the cut point → RecordPickerSheet opens in edit mode
  → step 1: label pick → step 2: catalog# pick → tap "SET CUT POINT"
      → validates via POS-01 + phantom check
      → on pass: bin-card "starts at" updates; change staged; PREVIEW CHANGES enables
      → on fail: validation error in sheet; sheet stays open
```

### State machine: override drift detection

```
[on boundary_cache load / admin edit commit] → compare all overrides to current autoFraction
  → for each where abs(override - auto) > DRIFT_THRESHOLD:
      → SegmentLegend chip switches to drifted state (caution icon + "review" text)
      → "reset to {N}%" tap action appears inline
  → tap "reset to {N}%": override fraction updated to autoFraction; chip returns to non-drifted OVERRIDE state; change staged
  → override never auto-changes (D-03)
```

### State machine: contiguity validation

```
[PREVIEW CHANGES tapped] → POST /api/admin/cubes/validate (dry-run)
  → contiguity pass: diff preview sheet opens
  → contiguity violation: diff preview BLOCKED; hard error banner in editor
      → copy: "This cut would split LABEL across non-adjacent bins. Reposition the cut or remove the existing boundary."
      → PREVIEW CHANGES button remains disabled until the violation is resolved
```
