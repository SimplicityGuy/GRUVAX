(BAK-01) and settings (BAK-02), atomic CSV/YAML boundary import (ADMN-05), and validated
settings import (BAK-02). All commit logic reuses the existing atomic infrastructure —

whitelisted PUT path; the PIN never leaves the DB (D-14).

load-bearing guarantees.

GREEN for export/import/settings.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin upload → import_.py | Untrusted multipart file (CSV/YAML) crosses into parse + DB write |
| import_.py → segment_overrides / boundary_history | Parsed data crosses into the shared gruvax schema |
| export.py → downloadable file | DB state crosses out to a file the owner may share |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-PIN-LEAK | Information Disclosure | export_settings | mitigate | SELECT WHERE key = ANY(_ALLOWED_SETTINGS_KEYS) — auth.pin_hash absent from the allowlist; unit test asserts no "pin_hash"/"auth." in output (D-14, Pitfall 12). Allowlist (not denylist) per security requirement. |
| T-07-YAML-BOMB | Tampering / DoS | import_boundaries / import_settings | mitigate | yaml.safe_load only (via Plan 02 parser + direct safe_load in settings import); upload size cap 100 KB → 413 |
| T-07-PARTIAL | Tampering | import_boundaries | mitigate | Validate ALL edits before ANY write; commit in a single transaction; on validation failure return 400 with zero writes (Pitfall 7). test_atomicity asserts unchanged state after a failed import. |
| T-07-SETTINGS-KEY | Elevation of Privilege | import_settings | mitigate | Reject auth.* (422 auth_key_rejected) and any non-allowlisted key (422 unknown_key) before any UPDATE; whole-file reject (no partial settings write) |
| T-07-CSRF | Spoofing | import POST endpoints | mitigate | require_admin enforces session + double-submit CSRF; export GETs are session-only (read-only, CSRF-exempt) |
| T-07-DOUBLE-COMMIT | Tampering (dup data) | import_boundaries | mitigate | Idempotency-Key header + idempotency_keys dedup (existing pattern) |
| T-07-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs (composition-only). No legitimacy checkpoint required. |
</threat_model>
