(`frontend/src/routes/admin/Settings.tsx:556-569`) use the CSS class
`settings-btn-secondary`, but that class has **no rule** in
`frontend/src/routes/admin/admin.css` — so the buttons fall back to default
browser styling (plain grey), breaking the Nordic Grid design language. The
sibling `SAVE LED SETTINGS` button (`.settings-btn-primary`) is styled correctly.

Add a `.settings-btn-secondary` rule that mirrors the established
`.editor-btn-secondary` convention (admin.css:891-920), adjusted to match the
full-width `.settings-btn-primary` LED-action stack. CSS-only — no TSX change.
</objective>
