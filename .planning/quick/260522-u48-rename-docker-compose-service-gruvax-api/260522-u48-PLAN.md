# Quick Task 260522-u48: Rename docker compose service `gruvax-api` → `api`

## Objective

Rename the Compose service key `gruvax-api` to `api` so the running container is
`gruvax-api-1` (project `gruvax` + service `api`) instead of the doubled-up
`gruvax-gruvax-api-1`. Shortens `docker compose exec gruvax-api …` to
`docker compose exec api …`.

## Why it's safe

Nothing `depends_on: gruvax-api` — `gruvax-api` is the depend<i>ent</i> (it depends on
`gruvax-dev-pg` + `mosquitto`), so no other service references it by name. The image
tag (`gruvax-api:local`) and the MQTT username default (`${MQTT_USERNAME:-gruvax-api}`)
are NOT the service name and stay unchanged.

## Tasks

1. **compose.yaml** — rename the service key `gruvax-api:` → `api:` (line 44).
   Update service-name references in the file's header comments
   (stack list + DB-connectivity + runbook notes) from `gruvax-api` to `api`.
   Leave `image: gruvax-api:local` and `MQTT_USERNAME` default untouched.

2. **README.md** — update the two `gruvax-api` service references
   (DB-connectivity note + project-tree compose description) to `api`.

3. **Verify** — `docker compose up -d` recreates the stack; confirm the container
   is now `gruvax-api-1` and `/api/health` returns 200.

## Done when

- `docker compose config` is valid and lists service `api`.
- `docker compose ps` shows container `gruvax-api-1` (not `gruvax-gruvax-api-1`).
- `curl localhost:8000/api/health` → 200.
