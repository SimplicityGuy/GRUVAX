# Quick Task 260519-p8t: Add design-language assets + rewrite README

## Objective

Commit the GRUVAX design-language package (now in `design/`) to the repo, and
rewrite the top-level `README.md` so its header follows the discogsography
README pattern — a centered `<div>` with a banner logo, a stack of badges, a
bold tagline, and a centered nav line — while applying the GRUVAX design
language (Nordic Grid identity; IKEA blue/yellow palette; Barlow Condensed /
Space Grotesk / DM Mono type system).

## Locked decisions (from user)

1. **Badges — static tooling only.** License, Python 3.13+, FastAPI, uv, just,
   Ruff, mypy, Docker, Claude Code. No CI-status badges (Build/Tests/codecov)
   until the corresponding GitHub Actions workflows exist — broken badges are
   worse than no badges.
2. **Logo — pixel-perfect PNG.** Render `design/gruvax-logo-banner.svg` via
   headless Chromium (loads the real Barlow Condensed web font) and commit the
   PNG, matching discogsography's PNG-banner approach. GitHub strips the SVG's
   Google-Fonts `@import` when an SVG is referenced from an `<img>`, so a raw
   SVG reference would fall back to Arial Narrow.
3. **Content — preserve, restyle header.** Keep all existing substantive README
   sections (How it works, Hardware, Stack, v1 Features, Repository layout,
   Planning artifacts, Status, License). Add the centered header treatment and
   thread the design-language vocabulary through, plus a short Design section
   pointing at `design/`.
4. **Commit gate — user reviews first.** User will review and hand-edit the
   README before any commit. Do NOT commit until the user approves.
