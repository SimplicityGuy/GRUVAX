<objective>
Fix the CI Build failure caused by `react-qr-code@2.1.1` (dependabot bump in commit 7a749b8). The package's `package.json` `exports["."]` map omits a `types` condition, so under `moduleResolution: bundler` TypeScript honors `exports` and cannot reach the shipped `types/index.d.ts`, producing TS7016 in `frontend/src/routes/kiosk/PairView.tsx`.

Purpose: Restore a green CI type-check gate without pinning the dependency back. Type-resolution-only fix.
Output: A local ambient declaration shim at `frontend/src/react-qr-code.d.ts` that mirrors the package's own shipped declaration, picked up automatically because `tsconfig.app.json` has `include: ["src"]`.
</objective>
