<lint_rule_mechanics>
The `react-hooks/set-state-in-effect` rule fires when a `useState` setter is called
**lexically inside an effect's setup function body** — including a bare `void asyncFn()`
call whose body eventually calls a setter. The rule does NOT fire for setters called inside:
  - an event handler (onClick / onChange / onDigit callback)
  - a `setInterval` / `setTimeout` callback passed to the timer
  - a `.then(...)` continuation that the rule cannot statically tie to the effect body
  - the render body itself (the "adjust state during render" escape hatch)
Every fix below relocates the flagged setState into one of those non-flagged positions
WITHOUT changing when the setState actually runs at runtime.
</lint_rule_mechanics>
