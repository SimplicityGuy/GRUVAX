## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser → /api/admin/profiles/* | PIN-gated mutations; the pasted PAT (secret) crosses here in the connect/rotate body. |
| GRUVAX → discogsography | Bearer PAT sent on the inline test-sync; must never be logged in plaintext. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05-01 | Spoofing | unauthenticated profile mutation | mitigate | Every POST/PATCH/DELETE under /api/admin/profiles requires require_admin (PIN session + CSRF double-submit). |
| T-02-05-02 | Information Disclosure | PAT plaintext at rest or in logs | mitigate | encrypt_pat (Fernet) before storage (carries from P1); the structlog dscg_* redactor masks Bearer tokens; GET /profiles never returns app_token_encrypted. |
| T-02-05-03 | Tampering | one discogsography user mapped to two profiles | mitigate | Connect captures user_id; 409 user_id_collision via the uq_profiles_dgs_user_id_active partial-unique index + explicit pre-check (D-09 strict match). Rotate requires same-user match. |
| T-02-05-04 | Tampering | background sync exception silently lost | mitigate | _run_sync_background catch-all + logger.exception; last_sync_status flips to 'failed' (Pitfall 3); the poll surfaces 'failed'. |
| T-02-05-05 | Denial of Service | a multi-second sync starving the pool | mitigate | Pitfall 6 pool discipline preserved (tight preflight checkout released before the long call); 202 returns before sync runs. |
| T-02-05-06 | Elevation of Privilege | stale browse binding to a soft-deleted profile | mitigate | Soft-delete pops the six registry entries; per-profile deps (Plan 02-02) then return 404 for the evicted profile (D2-03). |
| T-02-05-SC | Tampering | npm/pip installs | accept | No package installs (RESEARCH §Package Legitimacy Audit: none new). |

No HIGH-severity unmitigated threats.
