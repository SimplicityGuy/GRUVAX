### Surface 3: Connect / Sync Feedback States

#### Connect-PAT flow (in drawer)

**Step 1 вҖ” Idle:** PAT input + "CONNECT PAT" button (disabled if input empty).

**Step 2 вҖ” Test sync in progress (synchronous, ~1s):**
- Button changes to "CONNECTINGвҖҰ" (Barlow Condensed 700, same width)
- `aria-busy="true"` on button
- Inline spinner replaces button icon (20px Lucide `Loader2` spinning, `--gruvax-blue`)
- Input disabled

**Step 3a вҖ” Test sync success вҶ’ full sync kicks off:**
- Drawer stays open; button state transitions to "SYNCINGвҖҰ"
- Status badge on the card changes to SYNCING (animated pulse)
- Drawer shows sync progress section (see below)
- Drawer "CANCEL" button relabeled to "CLOSE" (sync continues in background)

**Step 3b вҖ” Test sync error (synchronous):**
- Error banner appears inline above the button (same `.sheet-error` class вҖ” red text, `role="alert"`)
- Input re-enabled so user can correct and retry
- Exact error messages вҖ” see Copywriting Contract

#### Sync-now flow (202 + poll)

- POST вҶ’ 202 вҶ’ poll `GET /api/admin/profiles/{id}` at 2s cadence
- Drawer shows a compact progress section:

### Accessibility

- All interactive elements: minimum 44Г—44px touch target
- All icon-only controls: `aria-label` required
- All error states: `role="alert"`
- All modals/drawers: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to heading, focus trapped on open, focus restored on close
- Status badges: descriptive `aria-label` on the badge element (e.g., `aria-label="Status: Connected"`)
- Profile picker cards: `role="button"` or `<button>` element, `aria-label="Choose {profile name} collection"`
- Syncing state: `aria-busy="true"` on the button/region being updated
- Toast: `role="status"` + `aria-live="polite"` (non-urgent, auto-dismisses)
- Profile deletion confirm: `aria-describedby` pointing to the warning body text
