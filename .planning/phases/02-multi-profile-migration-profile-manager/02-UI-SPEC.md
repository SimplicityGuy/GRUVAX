## Spacing Scale

Source: `design/gruvax-design-tokens.css` — `--gruvax-space-*` tokens. Use these exclusively;
never write raw pixel values.

> **Token basis:** Every value below is a `--gruvax-space-*` token defined in
> `design/gruvax-design-tokens.css`, and every one is a multiple of 4. `--gruvax-space-3` (12px)
> and `--gruvax-space-9` (96px) are project token values (both multiples of 4) defined in the
> token file, extending the abstract 8-point base set — they are not new declarations.

| Token | CSS Var | Value | Phase 2 Usage |
|-------|---------|-------|---------------|
| xs | `--gruvax-space-1` | 4px | Icon-to-text gaps, badge margins |
| sm | `--gruvax-space-2` | 8px | Compact element spacing |
| md-sm | `--gruvax-space-3` | 12px | Picker card internal padding, drawer section gaps |
| md | `--gruvax-space-4` | 16px | Default element spacing, sheet body horizontal padding |
| lg | `--gruvax-space-5` | 24px | Card vertical rhythm, form field gaps in drawer |
| xl | `--gruvax-space-6` | 32px | Profile list section gaps |
| 2xl | `--gruvax-space-7` | 48px | Bottom safe-area clearance in drawer actions |
| 3xl | `--gruvax-space-8` | 64px | Page-level vertical spacing |
| 4xl | `--gruvax-space-9` | 96px | Onboarding screen top offset |

All spacing-scale values above are multiples of 4.

**Touch-target minimums (accessibility floors, not spacing-scale entries):**
- Touch targets on profile picker cards: minimum 44px height enforced via `min-height: 44px` — matches `record-picker-row` pattern.
- Switch-profile corner button: 44×44px touch target minimum.

**Component density override (NOT part of the token spacing scale):**
- **Status badge pill** inherits the established v1.0 `.badge` pill internal padding from `admin.css`
  (the history/cube badge pattern). This is a component-level density inheritance for visual
  consistency with the existing v1.0 history and cube badges — it is **not** a spacing-scale value
  and introduces **no** new token. Phase 2 reuses the existing badge pill CSS rather than
  re-specifying its internal padding, so the multiple-of-4 spacing contract is unaffected.

### Surface 3: Connect / Sync Feedback States

#### Connect-PAT flow (in drawer)

**Step 1 — Idle:** PAT input + "CONNECT PAT" button (disabled if input empty).

**Step 2 — Test sync in progress (synchronous, ~1s):**
- Button changes to "CONNECTING…" (Barlow Condensed 700, same width)
- `aria-busy="true"` on button
- Inline spinner replaces button icon (20px Lucide `Loader2` spinning, `--gruvax-blue`)
- Input disabled

**Step 3a — Test sync success → full sync kicks off:**
- Drawer stays open; button state transitions to "SYNCING…"
- Status badge on the card changes to SYNCING (animated pulse)
- Drawer shows sync progress section (see below)
- Drawer dismiss button relabeled to "CLOSE" (sync continues in background)

**Step 3b — Test sync error (synchronous):**
- Error banner appears inline above the button (same `.sheet-error` class — red text, `role="alert"`)
- Input re-enabled so user can correct and retry
- Exact error messages — see Copywriting Contract

#### Sync-now flow (202 + poll)

- POST → 202 → poll `GET /api/admin/profiles/{id}` at 2s cadence
- Drawer shows a compact progress section:
  ```
  ┌─────────────────────────────────────────────┐
  │  Syncing…  [animated spinner]               │
  │  3,142 items processed                      │  ← DM Mono 14px, updates on each poll
  └─────────────────────────────────────────────┘
  ```
- Spinner: 20px animated circle using `animation: spin 1s linear infinite` on a `border-top` ring; ring color `--gruvax-yellow` on `--gruvax-blue-faint` background circle
- Item count: DM Mono 14px, `--gruvax-text-secondary`, updates on each poll result
- On poll returning `last_sync_status = 'ok'`:
  - Spinner disappears
  - Inline success message: "Sync complete — {N,###} records" (Space Grotesk 14px, `--gruvax-success`)
  - Card in background updates status badge to CONNECTED and item count
  - Toast notification (see below)
- On poll returning `last_sync_status = 'failed'`:
  - Inline error message (`.sheet-error`) — see Copywriting Contract

#### Toast notification (sync completion)

Appears at the top of the viewport (above all content, below admin topbar). Auto-dismisses after 4 seconds. No close button needed (auto-dismiss is sufficient for this low-stakes notification).

## Interaction Contract

### Animation

All new surfaces inherit the project animation system. No new easings; use existing tokens.

| Interaction | Duration | Easing | Token |
|-------------|----------|--------|-------|
| Drawer slide up | `var(--gruvax-duration-base)` | `var(--gruvax-ease-decelerate)` | Matches `sheet-slide-up` in admin.css |
| Card hover state | `var(--gruvax-duration-fast)` | `var(--gruvax-ease-standard)` | 150ms |
| Toast entry (slide-in from right) | `var(--gruvax-duration-base)` | `var(--gruvax-ease-decelerate)` | 250ms |
| Toast exit (fade out) | `var(--gruvax-duration-fast)` | `var(--gruvax-ease-standard)` | 150ms |
| Sync spinner rotation | `1s linear infinite` | n/a | Custom `@keyframes spin` |
| Syncing badge pulse | `1.5s ease-in-out infinite alternate` | n/a | Opacity 0.5→1 on badge background |
| Confirm modal entry | `var(--gruvax-duration-fast)` + scale(0.95→1) | `var(--gruvax-ease-spring)` | Spring pop |
| Switch-profile button press | `var(--gruvax-duration-fast)` bg change | `var(--gruvax-ease-standard)` | |

### Accessibility

- All interactive elements: minimum 44×44px touch target
- All icon-only controls: `aria-label` required (e.g. PAT show/hide toggle: `aria-label="Show token"` / `aria-label="Hide token"`)
- All error states: `role="alert"`
- All modals/drawers: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to heading, focus trapped on open, focus restored on close
- Status badges: descriptive `aria-label` on the badge element (e.g., `aria-label="Status: Connected"`)
- Profile picker cards: `role="button"` or `<button>` element, `aria-label="Choose {profile name} collection"`
- Syncing state: `aria-busy="true"` on the button/region being updated
- Toast: `role="status"` + `aria-live="polite"` (non-urgent, auto-dismisses)
- Profile deletion confirm: `aria-describedby` pointing to the warning body text

### State Machine: Profile Card in Admin List

```
PENDING
  └─ [Connect PAT] → SYNCING → CONNECTED
                   → error   → PENDING (back to form)

CONNECTED
  ├─ [Sync Now]   → SYNCING → CONNECTED
  │                         → CONNECTED (with failure message)
  ├─ [Rotate PAT] → SYNCING → CONNECTED
  │                         → CONNECTED (rotation failure)
  └─ [Delete]     → confirm → [deleted, removed from list]

SYNCING
  └─ [Close drawer] → drawer closes; card shows SYNCING badge + polls in background
```
