<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → per-profile endpoint | Untrusted path `profile_id` enters here; must be validated against the session cookie, never trusted as authoritative (D2-04). |
| sync background task → in-memory registry | A sync mutates only its own profile's registry slot. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Spoofing | path profile_id in `*_for_profile` deps | mitigate | Deps validate path profile_id == browse-binding cookie; 403 on mismatch, 400 on unbound (D2-04). Path param never authoritative. |
| T-02-02-02 | Information Disclosure | cross-profile cache reads via registry mis-key | mitigate | Registry resolved by str(profile_id) (Pitfall 2); distinct keys → distinct instances → isolation by construction (test_cache_registry asserts). |
| T-02-02-03 | Information Disclosure | collection_changed published before cache reload (stale read race) | mitigate | _refresh_profile_caches enforces invalidate→load→publish ordering (Pitfall A); source-order acceptance check. |
| T-02-02-04 | Elevation of Privilege | bus resolution dep reaching into the pool | mitigate | get_bus_for_profile reads only app.state (Pitfall 10 preserved); no get_pool dependency. |
| T-02-02-SC | Tampering | npm/pip installs | accept | No package installs (RESEARCH §Package Legitimacy Audit: none new). |

No HIGH-severity unmitigated threats.
