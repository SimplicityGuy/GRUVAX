new endpoints (GET segment data, POST overrides, PUT cut point, POST insert-cut), extend

new router. All endpoints carry `require_admin`, use `%s` parameterized SQL, and route every write
through the existing `validate → bulk → invalidate → publish` pipeline.

intent (drag override, edit cut, insert cut) has a server-authoritative, validated, atomic,
undoable backend — the contract the Wave 4 UI consumes. Reuses Phase 3 machinery wholesale (D-07).

Output: `segments.py` (new), extended `validation.py` + `cubes.py`, updated `router.py`, the
contiguity validator test, and the segment-API integration tests (stubbed in Plan 01) now passing.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/admin/cubes/* | Admin write endpoints; PIN-gated; request bodies cross here untrusted |
| admin request body → segment_overrides / cube_boundaries | label + fraction + cut values must be validated before persistence |
| cut-insert request → physical shelf | cascade must not exceed the physical cube count (resource exhaustion) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Tampering | POST overrides | mitigate | Server rejects (400) any override whose label is not in the bin's derived SegmentCache segments (phantom override injection); integration test asserts the 400 |
| T-05-08 | Tampering | override `fraction` | mitigate | Pydantic model bounds fraction to (0.0, 1.0]; DB CHECK from Migration 0005 is the storage backstop (defense in depth, V5) |
| T-05-09 | Denial of Service | POST insert-cut cascade | mitigate | validate_insert_cut counts physical cubes and rejects with shelf_overflow before any write (Pitfall 3 / D-06) |
| T-05-10 | Spoofing | Idempotency-Key replay of cut inserts | mitigate | Existing Phase 3 Idempotency-Key dedup table handles replays as no-ops; reused unchanged (D-07) |
| T-05-11 | Elevation of Privilege | all new admin endpoints | mitigate | `Depends(require_admin)` (V2/V4) on every new handler; unauthenticated 401/403 integration test per endpoint |
| T-05-12 | Tampering | SQL injection via label/catalog | mitigate | All new SQL uses `%s` parameterization (psycopg DB-API); no f-string interpolation of user input |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages this phase (RESEARCH Package Legitimacy Audit: none). No install tasks; no checkpoint required |
