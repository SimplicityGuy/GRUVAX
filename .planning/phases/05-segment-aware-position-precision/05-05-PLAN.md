<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser -> admin API | authenticated fetches (CSRF + session); request bodies originate client-side |
| DOM render of record/segment data | label/catalog strings rendered into the DOM; XSS risk if injected unsafely |
| drag input -> override fraction | client computes fractions; server re-validates (sum/label presence) on commit |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05-01 | Tampering | SegmentStrip / RecordPicker DOM build | mitigate | vanilla DOM via `el()` + `node.replaceChildren()`, never `innerHTML` (boundary-editing.md + CLAUDE.md security hook) — defeats DOM-injection XSS from label/catalog strings |
| T-05-05-02 | Tampering | client-computed override fractions | mitigate | server re-validates fraction bounds + label-in-bin on POST /overrides (05-04); client values are advisory only |
| T-05-05-03 | Spoofing / EoP | admin fetch without session | mitigate | reuse Phase 3 adminFetch (CSRF token + session cookie); 401 redirects to PIN; no new auth surface |
| T-05-05-04 | Tampering | contiguity / overflow bypass via UI | mitigate | UI hard-block is advisory; the server validators (05-04) are authoritative on the shared validate->commit path |
| T-05-05-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs this wave (RESEARCH.md: zero new deps; Lucide React already installed) |
</threat_model>
