<objective>
Build the reusable segment-editing primitives: the `el()` vanilla-DOM helper, the typed
`adminClient` calls for the four Phase 5 endpoints, the `SegmentStrip` (proportional bar with
sum-conserving pointer-capture drag-to-override handles, built with `el()` + `replaceChildren()`),
the `SegmentLegend` (AUTO/OVERRIDE chips with the 3pp drift review state + resync + straddle
captions), and the shared `RecordPickerSheet` (slide-up two-step autocomplete reusing Phase 3
phantom-block, surfacing duplicate/variant rows). All visuals consume `var(--gruvax-*)` tokens.

Purpose: These are the load-bearing UI primitives the full cut-point editor (Plan 05) assembles.
The drag interaction (SEG-04 width overrides) and the picker (SEG-08 cut editing) are the highest-
risk UI surfaces — building and verifying them in isolation keeps the assembly plan small. The
`el()` + `replaceChildren()` build constraint is a hard security requirement (no raw-HTML injection).

Output: `frontend/src/lib/dom.ts` (el helper), extended `adminClient.ts` + `types.ts`, three new
components (SegmentStrip, SegmentLegend, RecordPickerSheet), and their CSS in `admin.css` — all
consuming design tokens, no hardcoded hex.
</objective>
