# Phase 1 — UI Design Contract: First Search → Cube Highlight

> Visual and interaction contract for the kiosk search view: a 2×(4×4) grid of 32 cubes
> with row+col address overlays, a typeahead search box, a ranked results list, and
> primary-cube highlight on selection. Generated by gsd-ui-researcher.

## Typography

All roles reference named tokens from `gruvax-design-tokens.css`. Never hardcode hex or px values outside the token file.

> **Per-family discipline (multi-family design system).** GRUVAX uses a committed, LOCKED
> multi-family design system — Barlow Condensed (display/labels), Space Grotesk (UI body),
> and DM Mono (data) — mandated by `CLAUDE.md` and `design/gruvax-design-language.md`, which
> forbid inventing new styles or consolidating away the three-font system. Because each
> family carries a distinct, non-overlapping role, the standard "max 4 sizes / max 2 weights"
> typography discipline is applied **per font family, not globally**. The checker explicitly
> sanctioned this resolution path. Collapsing to 4 global sizes or dropping Barlow Condensed
> 900 would degrade the committed design language and is therefore out of bounds. Each family
> below stays within 4 sizes and 2 weights:
>
> | Font family | Sizes used (Phase 1) | Weights used (Phase 1) | Within per-family limit? |
> |-------------|----------------------|------------------------|--------------------------|
> | Barlow Condensed | 2 (12px, 24px) | 2 (700, 900) | Yes — 2 ≤ 4 sizes, 2 ≤ 2 weights |
> | Space Grotesk | 3 (14px, 16px, 18px) | 1 (400) | Yes — 3 ≤ 4 sizes, 1 ≤ 2 weights |
> | DM Mono | 1 (14px) | 1 (400) | Yes — 1 ≤ 4 sizes, 1 ≤ 2 weights |

### Barlow Condensed — Display (ALL CAPS labels, headings, shelf identifiers)

**2 sizes · 2 weights (700, 900) — within the per-family limit.**

| Role | Token | Size | Weight | Tracking | Line Height | Usage |
|------|-------|------|--------|----------|-------------|-------|
| Shelf label | `--gruvax-text-display-md` | 24px | 900 | `--gruvax-tracking-display` (0.02em) | `--gruvax-leading-tight` (1.1) | "SHELF A", "SHELF B" above each 4×4 grid |
| Address overlay | `--gruvax-text-caption` (display variant) | 12px | 700 | `--gruvax-tracking-label` (0.14em) | `--gruvax-leading-tight` (1.1) | Row+col label inside each cube (e.g. "A1") |

Barlow Condensed 900 is **retained** per the locked design language (the wordmark and all
display headings depend on it). The address overlay stays on Barlow Condensed 700 — it is not
moved to another family.

### Space Grotesk — UI Body (inputs, results, instructions)

**3 sizes · 1 weight (400) — within the per-family limit.**

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Search input text | `--gruvax-text-body-lg` | 18px | 400 | `--gruvax-leading-normal` (1.5) | User-typed query text |
| Search placeholder | `--gruvax-text-body-lg` | 18px | 400 | `--gruvax-leading-normal` (1.5) | Placeholder text (color: `--gruvax-text-faint`) |
| Result row — artist/title | `--gruvax-text-body` | 16px | 400 | `--gruvax-leading-normal` (1.5) | Primary line of each result |
| Result row — label | `--gruvax-text-body-sm` | 14px | 400 | `--gruvax-leading-normal` (1.5) | Secondary line of each result (label name) |
| No-results message | `--gruvax-text-body` | 16px | 400 | `--gruvax-leading-normal` (1.5) | Sentence case, `--gruvax-text-secondary` |

Space Grotesk 500 exists in the token file but is reserved for Phase 3 UI labels — not used in Phase 1, keeping this family at a single weight here.

### DM Mono — Data (catalog numbers)

**1 size · 1 weight (400) — within the per-family limit.**

| Role | Token | Size | Weight | Usage |
|------|-------|------|--------|-------|
| Catalog number in results | `--gruvax-text-mono` | 14px | 400 | Catalog# in result rows (fixed-width rendering) |

The 11px DM Mono role (`--gruvax-text-mono-sm`, catalog# inside the address overlay) is
**removed from the Phase 1 type scale**. Phase 1 uses 80px kiosk cells, so the catalog# is not
rendered inside cubes — only the row+col address overlay appears there. 11px DM Mono is
**reserved for future denser cell layouts** (e.g. the 40px admin/compact grid) and may
re-enter the scale in a later phase that uses those layouts.

### Per-family weight set for this phase

- **Barlow Condensed:** 700 (address overlay) + 900 (shelf identifiers) — 2 weights
- **Space Grotesk:** 400 (body) only — 1 weight (500 reserved for Phase 3 UI labels)
- **DM Mono:** 400 (data) only — 1 weight

Every family is within the per-family 4-size / 2-weight discipline, with full design-language
fidelity preserved (Barlow Condensed 900 retained, address overlay on Barlow Condensed).

Source: `design/gruvax-design-language.md` §Typography + `design/gruvax-design-tokens.css` (locked).

**Auto-selected (top result):** same selected state applied automatically on search result arrival.

### Top-result auto-highlight behavior

On search result arrival:
1. The results list appears (Framer Motion `AnimatePresence`, 200ms ease-decelerate).
2. The top result is immediately marked selected (left border accent, `--gruvax-blue-faint` row bg).
3. All cubes transition to `dim` state (CSS transition, `--gruvax-led-off-duration` 500ms, stagger 10ms per cell via JS delay array).
4. The primary cube for the top result transitions to `lit` state (CSS transition, `--gruvax-led-on-duration` 300ms, `--gruvax-ease-spring`).
5. LED glow pulse: scale 1→1.04→1, 600ms, CSS `@keyframes` — `transform: scale(1.04)` at 50%.

On tap of a different result row:
- Previous lit cube transitions to `dim` (LED off: 500ms smooth).
- New primary cube transitions to `lit` (LED on: 300ms spring).
- No other animation (Phase 1 — no label-span, no sub-cube bar).

On clear-X tap or empty query:
- All cubes transition to `dim` (staggered 10ms, 500ms smooth).
- Results list disappears (Framer Motion exit, 150ms).

**Interruption rule (SRCH-05 / CUBE-08 spirit):** Any new keystroke during an in-flight animation cancels the in-progress LED transition immediately and starts the new sequence. Use `gsap.killTweensOf(cubeEl)` or CSS `transition: none` swap pattern before applying new state.

## Empty State Contract

### Empty cube (CUBE-05)

A cube is in the `empty` state when its covering boundary range has `confidence: 0.0` (no boundary covers that label) OR when a cube has no boundary assignment at all.

| Property | Value |
|----------|-------|
| Fill | `--gruvax-cell-empty` (#F2F2F2) |
| Border | 1.5px dashed `#DDDDDD` |
| Address overlay | Rendered normally (`--gruvax-text-faint`, slightly dimmer than dim-state overlay) |
| No additional label needed | The desaturated + dashed border is the signal |

Empty cubes do not transition to `lit` — they remain `empty` even if somehow targeted. If a result returns `primary_cube: null`, no cube is highlighted; the address overlay on an empty cube reads "—" or is omitted.

### No-results state (SRCH-04)

When a search returns zero results and the query is non-empty:

| Element | Spec |
|---------|------|
| Results panel | Shown with a single row |
| Icon | 24px inline SVG magnifier with a slash, `--gruvax-warning` (#E6A800) |
| Heading | "No records found" — Space Grotesk 16px weight 400, `--gruvax-text-secondary` |
| Body | "Try a different search or check the label name." — Space Grotesk 14px weight 400, `--gruvax-text-muted` |
| Grid cubes | All remain in `dim` or `empty` state; no cube highlights |

## Animation Contract

All transitions use tokens from `gruvax-design-tokens.css`. Do not author ad-hoc easings.

### Cube LED on (dim → lit)

```css
transition:
  background var(--gruvax-led-on-duration) var(--gruvax-led-on-ease),
  box-shadow  var(--gruvax-led-on-duration) var(--gruvax-led-on-ease);
/* 300ms cubic-bezier(0.34, 1.56, 0.64, 1) — spring pop with overshoot */
```

### Cube LED off (lit → dim)

```css
transition:
  background var(--gruvax-led-off-duration) var(--gruvax-led-off-ease),
  box-shadow  var(--gruvax-led-off-duration) var(--gruvax-led-off-ease);
/* 500ms cubic-bezier(0.4, 0.0, 0.2, 1) — smooth fade */
```

### Full search-result reveal sequence

| Step | What | Duration | Easing Token |
|------|------|----------|--------------|
| 1 | Results list slides in (height + opacity) | 200ms | `--gruvax-ease-decelerate` |
| 2 | All cubes → dim (staggered, 10ms per cell) | 500ms fade | `--gruvax-led-off-ease` |
| 3 | Primary cube → lit (after step 2 completes, 50ms delay) | 300ms spring | `--gruvax-led-on-ease` |
| 4 | LED glow pulse (scale 1 → 1.04 → 1) | 600ms | `--gruvax-ease-spring` |

**Total sequence budget:** ≤ 850ms from data receipt to animation complete (200 + 500 + 50 + 300 + header overlap = ~600ms perceived, fits within the design language's "LED physics" intent).

### Results list enter/exit (Framer Motion)

Use `AnimatePresence` with:
- Enter: `initial={{ opacity: 0, y: -8 }}`, `animate={{ opacity: 1, y: 0 }}`, duration 200ms `--gruvax-ease-decelerate`
- Exit: `exit={{ opacity: 0, y: -8 }}`, duration 150ms `--gruvax-ease-accelerate`

### General UI feedback

All non-LED interactions (hover, focus ring, row selection, clear-X tap) complete within 150ms using `--gruvax-duration-fast` (`150ms`) + `--gruvax-ease-standard`.
