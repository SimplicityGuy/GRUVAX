<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/client → /api/* | untrusted query params (`q`, `release_id`, `limit`) cross here |
| gruvax-api → Postgres | parameterized queries only; read-only on v_collection |
| gruvax-api → mosquitto | publish-only hello; broker on internal network (no host port) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Tampering | SQL injection via search `q` | mitigate | psycopg `$1` placeholders only; no f-string SQL; SQLi payload integration test (Task 2) |
| T-01-08 | Denial of Service | unbounded `limit` | mitigate | `limit: int = Query(ge=1, le=50)` validated at router (422 on overflow) |
| T-01-09 | Tampering | non-integer `release_id` | mitigate | `release_id: int` typed param; Pydantic returns 422 |
| T-01-10 | Denial of Service | oversized `q` slow FTS | mitigate | `q: str = Query(max_length=200)` capped at router |
| T-01-11 | Denial of Service | mosquitto unreachable blocks API | mitigate | MQTT connect wrapped in try/except in lifespan; never blocks request path (DEP-01) |
| T-01-SC | Tampering | aiomqtt/dependency install | accept | Pinned, registry-verified per RESEARCH audit; no [SUS]/[SLOP] packages |
</threat_model>
