### Fill shading palette (this phase — decision D-01)

The mini-Kallax fill shading uses **blue-family saturation only**. Yellow is excluded per
the design language reservation rule: yellow = active/changed/lit (LED state and the
edited-bin highlight `.locator-cell--lit`). Overloading yellow for occupancy would collide
with that meaning.

| Cube state | Background token | Border style | Notes |
|------------|-----------------|--------------|-------|
| Empty (`is_empty: true`) | `--gruvax-cell-empty` (#F2F2F2) | `1.5px dashed --gruvax-cell-empty-border` | CUBE-05 desaturated state — UX-01 SC 1 |
| 0% fill (non-empty, fill_level = 0) | `--gruvax-cell-dim` (#D8E8F5) | `1.5px solid --gruvax-cell-dim-border` | Base dim state |
| fill_level 0–1.0 (continuous) | `color-mix(in srgb, var(--gruvax-blue) calc(var(--fill) * 70%), var(--gruvax-cell-dim))` | `1.5px solid --gruvax-cell-dim-border` | `--fill` = clamped fill_level [0,1]; see gradient section |
| Full (fill_level ≥ 1.0, clamped) | `var(--gruvax-blue)` | `1.5px solid --gruvax-blue-dark` | Deepest blue — visually distinct from empty (UX-01 SC 3) |
| Lit / edited bin (yellow — existing) | `--gruvax-cell-lit` (#FFDA00) | `--gruvax-cell-lit-border` | Unchanged from existing `.locator-cell--lit` — mutually exclusive with fill shading |

**Fill gradient technique (Claude's Discretion — recommended):**

Use a CSS custom property `--fill` (value: clamped `fill_level` in [0, 1]) set inline on
each cell element. The cell background is:

```css
/* On each .locator-cell[data-state="fill"] */
background: color-mix(
  in srgb,
  var(--gruvax-blue) calc(var(--fill) * 70%),
  var(--gruvax-cell-dim)
);
```

This maps:
- `fill_level = 0` → `--gruvax-cell-dim` (#D8E8F5) — pale blue, same as today's dim
- `fill_level = 0.5` → mid-blue (~35% `--gruvax-blue` mixed into dim)
- `fill_level = 1.0` → 70% `--gruvax-blue` mix — clearly saturated blue, distinct from empty gray

The 70% cap is intentional: pure `--gruvax-blue` (100%) is reserved for borders and text;
a 70% mix in the dim base reads as "full" without competing with chrome elements.

`color-mix(in srgb, ...)` is supported in Chromium 111+ and is the Chromium kiosk target.
No polyfill needed.

Alternatively, `opacity` layered over `--gruvax-cell-dim` with a `--gruvax-blue` overlay
at `opacity: calc(var(--fill) * 0.7)` achieves the same visual — prefer `color-mix` as it
is a single paint operation.

**Over-capacity (D-03):** Clamp `fill_level` to `1.0` visually. No distinct over-capacity
color, border, or texture. `fill_level > 1.0` renders identically to `fill_level = 1.0`.

## Component Inventory

### Modified: `LocatorHeader.tsx`

**Current state:** Purely display, non-interactive, 64 lines. Accepts `row` / `col` to
highlight the edited bin yellow. Renders `.locator-cell--lit` (yellow) or `.locator-cell--dim`
(pale blue) per cell.

**Phase 10 changes:**

1. **New prop: `cubes?: AdminCube[]`** — the full cube list for this unit, passed down from
   `ShelfBinList`. When absent the component falls back to today's dim/lit behavior
   (backward-compatible).

2. **Per-cell fill shading:** When `cubes` is provided, each cell resolves its `AdminCube`
   from the array (by `row + col`). Apply fill shading via inline `style`:
   - `is_empty: true` → class `locator-cell--empty`
   - Otherwise → inline `--fill: ${Math.min(fillLevel, 1)}` custom property + class
     `locator-cell--fill`
   - The lit (edited-bin yellow) state remains unchanged: class `locator-cell--lit` takes
     priority when `r === row && c === col && row !== -1`.

3. **Interactive tap target:** Each cell becomes a `<button type="button">` (or `role="button"`
   on a `<div>`) with `aria-label` containing the bin position and fill summary. Minimum touch
   target: 28px cell = inherent size; acceptable at this scale because the popover is
   supplemental detail, not a primary action. Do NOT add artificial padding that would break
   the grid geometry.

4. **Tap-to-reveal popover (D-05, D-06):** On tap, show an inline Nordic-Grid popover
   anchored to the tapped cell. One popover visible at a time; tapping another cell moves it;
   tapping the same cell or outside dismisses it. Dismiss also on `Escape` key.

## Interaction Contract

### Cell tap → popover

| Trigger | Action |
|---------|--------|
| First tap on any mini-cell | Show popover anchored to that cell; record which cell is active |
| Tap on same cell again | Dismiss popover |
| Tap on different cell | Move popover to new cell |
| Tap outside the mini-Kallax grid | Dismiss popover |
| `Escape` keydown | Dismiss popover |

**Popover content:**
- Non-empty cube: `{BIN_ID} · {record_count} records · {pct}%`
  where `BIN_ID` is the 1-based human position (e.g. "A1" = row 0, col 0 of unit 0;
  derive from `shelfLetter(unitId)` + `(row * 4 + col + 1)`), and `pct` = `Math.min(fill_level, 1) * 100`
  rounded to the nearest integer.
- Empty cube: `{BIN_ID} · Empty bin`
- Example (from CONTEXT.md D-06): `A1 · 47 records · 78%`

**Popover positioning:** Anchored below the tapped cell when there is room; flip to above
when the cell is in the bottom half of the grid. Horizontal: left-align to cell, clamped
to keep the popover within the `.locator-mini-grid` bounds. Use `position: absolute` on a
container wrapping the grid (not a portal — keeps the component self-contained per D-06).

### Transition

Fill shading transition on data refresh: `background var(--gruvax-duration-base) var(--gruvax-ease-standard)` (250ms).
This matches the app's standard UI feedback timing. Do NOT use LED spring transitions
(`--gruvax-led-on-ease`) for occupancy changes — LED physics are reserved for active/found
cube state.

Popover mount: instant (no animation). Popover dismiss: instant. The popover is supplemental
data, not a state transition.

## Live Update Contract (UX-01 SC 2)

The admin-route SSE listener must invalidate `['admin','cubes']` on:

1. `collection_changed` — nightly/manual sync changes record counts → fill_level changes
2. `boundary_changed` — admin boundary edit changes which records fall in each cube → fill_level changes

Both invalidations cause TanStack Query to refetch `adminGetCubes()`, which updates the
`cubes` prop passed to `LocatorHeader`, which triggers CSS re-render of fill shading via
the `--fill` custom property. No page reload required (UX-01 SC 2 confirmed).

**Current gap (CONTEXT.md D-04):** The kiosk `KioskView.tsx` SSE handlers (`boundary_changed`
at line 360, `collection_changed` at line 424) deliberately do NOT touch `['admin','cubes']`
(D-08 separation). The admin-route SSE listener must be located and extended — or a new
`useAdminSse` hook added if the admin route currently has no SSE consumer.

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Mini-Kallax ARIA label | Existing: `aria-label="Mini Kallax — edited bin at row X, col Y"` on `.locator-mini-grid` |
| Each cell ARIA label | `aria-label="{BIN_ID}: {N} records, {P}% full"` or `"{BIN_ID}: empty"` |
| Cell as interactive element | `<button type="button">` — receives focus, responds to Enter/Space |
| Popover accessible name | `role="tooltip"` with `aria-live="polite"` on the popover region |
| Color as sole indicator | Fill shading conveys fill level — must pair with ARIA label text |

**Accessibility note:** Yellow-on-blue and blue-on-yellow are ~3.1:1 (design language
constraint). The fill shading is decorative/supplemental; the popover provides the precise
numeric value. The cell ARIA label carries the full accessible content.
