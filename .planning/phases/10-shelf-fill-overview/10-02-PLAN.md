## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser <- SSE stream | The admin EventSource subscribes to /api/events/{boundProfileId} (same-origin, admin context). It reacts only to event TYPE (collection_changed / boundary_changed) by invalidating a local query — it does not parse or render event payload data. |

This phase adds NO new endpoints and NO new server input. The new admin SSE listener is a
read-only, same-origin, event-driven cache invalidator. No new high-severity attack surface.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-04 | Tampering | admin EventSource event payloads | mitigate | The hook reacts only to event type; it invalidates a query rather than rendering payload data, so a malformed/forged payload cannot inject content. Connection is same-origin to the existing per-profile SSE channel (already used by the kiosk). |
| T-10-05 | DoS (connection leak) | admin EventSource lifecycle | mitigate | useEffect cleanup calls es.close() on unmount (short-lived, scoped to ShelfBinList) — prevents accumulating connections across admin navigation. Asserted by the unmount test. |
| T-10-06 | Information Disclosure | /api/events/{profileId} subscription | accept | Same per-profile channel the kiosk already consumes; admin is PIN-gated on a LAN-only deployment. No new exposure. |
| T-10-SC | Tampering (supply chain) | npm installs | accept | No new packages this phase (RESEARCH: Package Legitimacy Audit not applicable). No blocking-human legitimacy checkpoint required. |
