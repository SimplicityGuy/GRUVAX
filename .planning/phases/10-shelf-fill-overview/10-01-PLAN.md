<objective>
Make the admin LocatorHeader mini 4x4 Kallax shade each cube by occupancy: a continuous
blue-saturation gradient for filled cubes, the CUBE-05 desaturated state for empty cubes, and
a tap-to-reveal popover showing the exact record count. This delivers UX-01 success criteria
1 (token-only fill shading) and 3 (empty vs full visually distinct), plus locked decisions
D-01 (blue not yellow), D-02 (continuous, not buckets), D-03 (clamp at full), D-05 (tap to
reveal), and D-06 (inline popover).

Purpose: The owner gets an instant visual of how full each bin is without opening the full
boundary editor — the milestone-closing UX polish of v2.1.
Output: An updated AdminCube type, a fill-shaded interactive LocatorHeader, new token-only
CSS classes, and a full render/interaction test suite.

This plan is frontend-only. No backend work — GET /api/admin/cubes already returns is_empty,
fill_level, and record_count (cubes.py:166-218). Do NOT add or modify any backend code.
</objective>
