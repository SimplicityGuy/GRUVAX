<objective>
SRCH-09 + PRIV-01 + PRIV-04: a session-only recently-pulled chip strip on the kiosk, a
no-PIN client-side-only "Reset kiosk" affordance, and a ~15-minute idle timeout that
returns the kiosk to its resting screen. The recently-pulled list is sessionStorage-backed
(survives an accidental reload, clears on tab close / hard Chromium restart per D-13) and is
explicitly excluded from any localStorage persist (PRIV-01). Reset and idle both clear the
search + chip list client-side only with zero API calls (L-05/D-09) while the device stays
paired and the bound profile stays selected.

Purpose: Privacy hygiene for a shared household kiosk — the previous visitor's lookups never
linger or persist across a restart.

Output: new store + idle hook + two kiosk components, wired into KioskView with token-only CSS.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| visitor A → visitor B (shared kiosk) | Search history must not leak between visitors or across a restart |
| kiosk client → backend | Reset/idle must NOT cross this boundary (zero API calls; client-only) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-PR-01 | Information Disclosure | recently-pulled persistence | mitigate | D-13: sessionStorage key `gruvax-kiosk-recent`, excluded from `gruvax-admin` localStorage; clears on hard Chromium restart (verified by hard-restart checkpoint step) |
| T-08-PR-02 | Information Disclosure | history lingering after a visit | mitigate | D-14/D-15: ~15-min idle timeout clears search + chips to resting screen |
| T-08-PR-03 | Elevation of Privilege | Reset performing a privileged action | mitigate | L-05/D-09: Reset is client-side only — zero API calls, no device unbind; checkpoint verifies empty Network tab |
| T-08-PR-04 | Spoofing | Reset hidden/shown by a server-wide flag (wrong owner) | mitigate | D-10: visibility driven by per-browser in-memory `adminStore.isLoggedIn`, never a server-wide admin_active flag |
</threat_model>
