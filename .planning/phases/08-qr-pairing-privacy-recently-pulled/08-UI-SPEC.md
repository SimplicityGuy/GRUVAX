| Text muted | `--gruvax-text-muted` | #777777 | QR caption; countdown (existing); idle prompt |
| Overlay scrim | `--gruvax-overlay-scrim` | rgba(0,40,85,0.55) | Confirm dialog modal scrim |
| Destructive | `--gruvax-error` | #C0392B | Reset kiosk button text (de-emphasis variant — see Reset Surface below) |

**Accent (`--gruvax-yellow`) is reserved for:**

1. The LED glow (`--gruvax-shadow-led`) on lit Kallax cells — unchanged from existing behavior.
2. The search-box focus ring (`outline: 2px solid var(--gruvax-yellow)`) — unchanged from existing behavior.
3. The chip active border when a chip is tapped and the re-locate animation fires (spring on).
4. The PairView code-card `--success` border flash on new code reroll — unchanged.
5. The "PAIR THIS DEVICE" button on the admin bind confirm screen receives yellow as its CTA background when the phone is logged in (`background: var(--gruvax-yellow); color: var(--gruvax-blue)`). This is large text (16px Barlow Condensed 700) — yellow-on-blue at 3.1:1 is acceptable at this size.

**Accent is NOT used for:** chip backgrounds, confirm dialog backgrounds, the Reset button, body text, or any decorative purpose.

**Accessibility constraints honored:**

- Blue-on-white: 7.2:1 (AAA). Safe for all text.
- Yellow-on-blue / blue-on-yellow: 3.1:1. Restricted to large text (18px+) or the CTA button (16px Barlow Condensed 700, visually equivalent to large text due to condensed heavy weight). Never used for body copy.
- QR code: `fgColor: var(--gruvax-blue)` on `bgColor: var(--gruvax-white)`. 7.2:1 contrast ratio — exceeds ISO/IEC 18004 minimum for module-to-background contrast.

### Surface 1: QR Code Block in PairView

**File:** `frontend/src/routes/kiosk/PairView.tsx` + `pair.css`

**Layout slot:** Below the existing `.pair-code-card` (the 4-digit digit block), centered on the vertical axis of the page. The QR block is a new `div.pair-qr-container` inserted below `.pair-code-card` and above `.pair-status-row`.

**Visibility:** Rendered when `pairingCode` is non-null AND `pairStatus` is not `'paired'` AND not `'loading'`. Hidden (unmounted) when the pairing is complete or the code has expired and is rerolling.

**QR dimensions and ECC:**

- Size: `160px` × `160px`. At a 7-inch display (1024×600 or 800×480) viewed from 50–80 cm, 160px renders modules at approximately 1.4–2.2 mm each, sufficient for reliable scan with a phone camera. Larger would push PairView content off-screen on 800×480.
- Error Correction Level: `"M"` (15% data restoration). Level Q (25%) would increase the QR version and reduce module size at 160px; M is the correct tradeoff at this scanning distance and screen size.
- `bgColor`: `var(--gruvax-white)` — white is the quiet zone background.
- `fgColor`: `var(--gruvax-blue)` — #0051A2, 7.2:1 contrast against white, exceeds scanner requirements.
- Quiet zone: the `react-qr-code` SVG renders a built-in quiet zone. The `.pair-qr-container` adds `padding: var(--gruvax-space-3)` (12px) on all sides to extend the quiet zone against any surrounding background.

**Caption:** `p.pair-qr-caption` rendered below the SVG, inside `.pair-qr-container`.

- Copy: `OR SCAN WITH PHONE` (ALL CAPS, Barlow Condensed 700, 13px, `--gruvax-tracking-descriptor` 0.20em, color `--gruvax-text-muted`)
- Margin-top: `var(--gruvax-space-2)` (8px) between SVG and caption

**Container styling:**

```css
.pair-qr-container {
  display: flex;
  flex-direction: column;
  align-items: center;
  padding: var(--gruvax-space-3);       /* 12px quiet-zone extension */
  background: var(--gruvax-white);
  border: 1.5px solid var(--gruvax-border-light);
  border-radius: var(--gruvax-radius-md);
  margin-top: var(--gruvax-space-3);    /* 12px from .pair-code-card */
  margin-bottom: var(--gruvax-space-5); /* 24px before .pair-status-row */
}

.pair-qr-caption {
  font-family: var(--gruvax-font-display);
  font-size: 13px;
  font-weight: 700;
  letter-spacing: var(--gruvax-tracking-descriptor); /* 0.20em */
  color: var(--gruvax-text-muted);
  text-transform: uppercase;
  margin: var(--gruvax-space-2) 0 0 0;
  line-height: var(--gruvax-leading-tight);
}
```

**Motion:** None. The QR SVG re-renders reactively when `pairingCode.code` changes (reroll). No animation needed — the digit card already handles the reroll flash. The QR is a silent information update.

**Accessibility:** `div.pair-qr-container` carries `aria-label="Scan QR code to pair this device"`. The QR SVG is decorative (the text code is the accessible path) — `role="img" aria-hidden="true"` on the `<QRCode>` output wrapper.

**File:** `frontend/src/routes/kiosk/KioskView.tsx` + new `frontend/src/routes/kiosk/RecentlyPulledStrip.tsx` + `kiosk.css`

**Layout slot:** A horizontal strip inside `.kiosk-content`, below the `.shelf-area` (the ShelfGrid + ShelfLabel blocks) and above the page bottom padding. The strip is `position: relative`, not sticky. It is `display: none` (unmounted) when `items.length === 0`.

**Strip container:**

```css
.recently-pulled-strip {
  margin-top: var(--gruvax-space-5);   /* 24px from shelf area */
  padding: 0 var(--gruvax-space-4);    /* 16px horizontal edge padding */
  display: flex;
  flex-direction: column;
  gap: var(--gruvax-space-2);          /* 8px between label row and chip row */
}

.recently-pulled-strip__label {
  font-family: var(--gruvax-font-display);
  font-size: 13px;
  font-weight: 700;
  letter-spacing: var(--gruvax-tracking-descriptor); /* 0.20em */
  color: var(--gruvax-text-muted);
  text-transform: uppercase;
  line-height: var(--gruvax-leading-tight);
}

.recently-pulled-strip__chips {
  display: flex;
  flex-direction: row;
  gap: var(--gruvax-space-3);          /* 12px between chips */
  overflow-x: auto;

  -webkit-overflow-scrolling: touch;
  padding-bottom: var(--gruvax-space-1); /* 4px to prevent chip shadow clipping */
}

.recently-pulled-strip__chips::-webkit-scrollbar {
  display: none;
}
```

**Strip label:** `RECENTLY PULLED` — Barlow Condensed 700, 13px, ALL CAPS, `--gruvax-tracking-descriptor` 0.20em, color `--gruvax-text-muted`.

**Chip dimensions:**

- Minimum height: 48px (touch target for finger use on 7" display)
- Width: `auto` (content-sized), minimum `88px`, maximum `200px` via `max-width`
- The chip is a `button` element (tappable, keyboard-navigable)

**Chip anatomy:** Two-line layout (stacked vertically, left-aligned):

- Line 1: `artist – title` truncated — Space Grotesk 14px weight 400, color `--gruvax-text-secondary`, `white-space: nowrap; overflow: hidden; text-overflow: ellipsis`
- Line 2: catalog number — DM Mono 14px weight 400, color `--gruvax-text-muted`, `white-space: nowrap; overflow: hidden; text-overflow: ellipsis`

**Chip content data model:** `{ release_id: number, title: string, artist: string, catalog_number: string }`. The chip renders `artist + " – " + title` on line 1 (if artist is empty, render `title` only). Line 1 max-width truncation applies before the chip max-width is reached.

**Truncation rules (7" display):**

- Chip `max-width: 200px`. At 14px Space Grotesk, approximately 22 characters fit before ellipsis. This is sufficient for most artist + dash + short title combinations.
- If `artist + " – " + title` would exceed the chip width, truncate at the combined string level (do not truncate artist and title independently).
- Catalog number line: DM Mono is monospaced — at 14px a chip 200px wide can display approximately 14 characters. Most catalog numbers fit; longer ones truncate gracefully.

**Chip styling:**

```css
.recently-pulled-chip {

  flex-direction: column;

  min-height: 48px;
  min-width: 88px;

  padding: var(--gruvax-space-2) var(--gruvax-space-3); /* 8px 12px */

  cursor: pointer;
  flex-shrink: 0;
  transition:

    border-color var(--gruvax-led-on-duration) var(--gruvax-led-on-ease);
}

.recently-pulled-chip:hover,
.recently-pulled-chip:focus-visible {
  background: var(--gruvax-blue-faint);
  border-color: var(--gruvax-blue);
  outline: none;
}

.recently-pulled-chip:active {
  background: var(--gruvax-blue-light);
  border-color: var(--gruvax-yellow);  /* LED physics: yellow on tap */
  transition:
    background  var(--gruvax-duration-fast) var(--gruvax-ease-standard),
    border-color var(--gruvax-led-on-duration) var(--gruvax-led-on-ease);
}

.recently-pulled-chip__title {
  font-family: var(--gruvax-font-ui);
  font-size: var(--gruvax-text-body-sm); /* 14px */
  font-weight: 400;
  color: var(--gruvax-text-secondary);
  line-height: var(--gruvax-leading-normal);
  white-space: nowrap;

}

  font-family: var(--gruvax-font-mono);
  font-size: var(--gruvax-text-mono); /* 14px */
  font-weight: 400;
  color: var(--gruvax-text-muted);
  line-height: var(--gruvax-leading-normal);
  white-space: nowrap;

}
```

**Enter/exit motion:** The chip strip enters with an opacity + translateY transition when it first becomes non-empty (first successful locate), exits when cleared by Reset or idle. Use CSS:

```css
.recently-pulled-strip {
  opacity: 0;
  transform: translateY(8px);
  animation: strip-enter var(--gruvax-duration-slow) var(--gruvax-ease-decelerate) forwards;
}

@keyframes strip-enter {
  to { opacity: 1; transform: translateY(0); }
}
```

The strip does not animate out — it is unmounted (opacity transition is implicit via React unmount). This matches the "fast feedback, under 150ms" philosophy: clearing is instant.

**New chip entering the strip:** When a new chip is prepended (most-recent-first), it slides in from the left using a brief `translateX(-12px) → translateX(0)` at `var(--gruvax-duration-fast)` (150ms). Existing chips shift right via CSS `flex` reflow. No GSAP needed — CSS transition on the new chip element.

**Overflow at 8 chips:** When the strip holds 8 chips (cap per D-07), the `overflow-x: auto` rule on `.recently-pulled-strip__chips` handles overflow. No visual indicator for scrollability — the kiosk user can swipe horizontally. On a 7" display at 800px width with 12px gaps, approximately 4 chips are visible at once; 8 chips require a horizontal scroll of ~400px.

**Interaction:** Tapping a chip calls the locate action for `release_id` (re-runs the TanStack Query locate query via `setSelectedReleaseId(item.release_id)`), which re-highlights the correct cube. No navigation occurs — the chip strip stays visible after a re-locate tap.

**Accessibility:** Each chip is a `<button>` with `aria-label={`${artist} – ${title}, catalog number ${catalog_number}`}`. The strip container has `aria-label="Recently pulled records"` and `role="list"` (each chip is `role="listitem"`).

### Surface 5: Resting Screen and Idle Return

**File:** `frontend/src/routes/kiosk/KioskView.tsx` + `kiosk.css`

**Definition:** The resting state is the empty search-first KioskView state — the same visual as a fresh page load with no query, no results, and no cube highlight. It is NOT a distinct route or fullscreen screensaver. The design language spec's screensaver (album artwork fullbleed, 3-min idle) is a separate future feature; Phase 8's idle behavior is a privacy/session clear, not a visual screensaver.

**What clears on idle/session end:**

- `useGruvaxStore.getState().clearSearch()` — clears query, results, cube highlight
- `useRecentlyPulledStore.getState().clear()` — clears chip strip
- Device pairing state is NOT cleared (D-15)
- The kiosk remains at the `/` route

**Visual result of clearing:** The kiosk shows the search box (empty), the shelf grid (all cells dim), and no chip strip. This is already the correct resting visual from prior phases.

**Idle timeout:** 15 minutes (`15 * 60 * 1000` ms), implemented via `useIdleTimer` hook (see 08-RESEARCH.md §Pattern 3). Reset events: `pointermove`, `pointerdown`, `keydown`, `touchstart`.

**Idle transition:** When the idle timeout fires, the clear happens immediately (no animation delay). The shelf grid cells transition to dim via their existing `--gruvax-led-off-duration` (500ms) smooth fade — the LED-physics off transition is already in place for clearSearch. The chip strip unmounts (instant). This is intentionally fast — a privacy action, not a cinematic one.

**Wake from resting:** Any touch or key input resets the idle timer. If the resting state was reached by an idle timeout, the user simply starts typing — the search box is still the primary interaction target and retains focus. No "tap to wake" intercept is needed.

**Idle indicator (optional, builder discretion):** No visual countdown or "screen will clear" warning is required for Phase 8. The 15-minute window is long enough that a warning adds complexity without proportional value. Revisit in a later phase if user testing reveals confusion.

### Surface 6: Admin Bind Prefill Confirm (Mobile-First)

**File:** `frontend/src/routes/admin/DeviceDrawer.tsx` (modified) + `frontend/src/routes/admin/admin.css`

**Context:** The phone user has scanned the QR, was prompted for the PIN (AdminShell modal overlay), entered it, and now sees the `/admin/devices` page with `?code=XXXX` already consumed. `DevicesManager` has called `setDrawerTarget('bind')` with `prefillCode` prop set to the 4-digit string.

**Trigger state:** `DeviceDrawer` in bind mode receives `prefillCode?: string`. When `prefillCode` is present, the drawer renders the **prefill confirm screen** instead of the NumericKeypad (D-04 — no auto-submit).

**Prefill confirm screen layout (mobile-first, 390px baseline):**

The drawer content replaces the NumericKeypad with:

1. **Instruction** (sentence case, Space Grotesk 16px weight 400, color `--gruvax-text-secondary`):
   `Pair this device with your GRUVAX collection using code:`

2. **Code display:** The 4-digit code rendered in DM Mono — the same `.pair-code-digit` size (clamp 48px–64px on phone) or a fixed `48px` at the drawer's mobile width. Color `--gruvax-text-primary`. Centered.

3. **CTA button ("PAIR THIS DEVICE"):**
   - Full-width, min-height 56px (primary action on phone)
   - Background: `--gruvax-yellow` (#FFDA00)
   - Text: `--gruvax-blue` (#0051A2) — "PAIR THIS DEVICE", Barlow Condensed 700, 16px, `--gruvax-tracking-label` 0.14em, ALL CAPS
   - Yellow-on-blue at 3.1:1 is acceptable here: 16px Barlow Condensed 700 is visually large and is the single prominent action on the screen. This matches the existing GRUVAX CTA pattern.
   - `border-radius: var(--gruvax-radius-md)` (8px)
   - On tap: calls `handleBind(prefillCode)`. After success, the drawer closes with the existing success transition.

4. **Secondary link ("Enter a different code"):**
   - Below the CTA, text-only link style
   - Space Grotesk 14px weight 400, color `--gruvax-text-secondary`, underline on hover
   - Tapping this switches the drawer back to the NumericKeypad view (clears `prefillCode`)
   - Touch target: min-height 44px via padding

**Copy rationale:** "PAIR THIS DEVICE" is the single-tap confirm required by D-04. The code is displayed above so the user can visually confirm the 4-digit code matches the kiosk before tapping. "Enter a different code" handles the edge case where the user scanned an old QR or wants the typed flow.

**Drawer background / scrim:** Uses existing `.device-drawer` styling — no changes to the drawer shell.

**Accessibility:** The CTA button has `aria-label="Pair this device using code {code}"` to make the code explicit in the accessible name.
