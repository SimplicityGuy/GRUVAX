<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| kiosk request → per-profile resolution | profile_id derived from binding server-side, never trusted from client |
| revoked device → per-profile endpoint | must be denied regardless of SSE liveness |
| SSE stream → DB pool | long-lived connection must not hold a pool slot |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Elevation of Privilege | client-supplied profile_id on per-profile request | mitigate | resolve_profile_from_request derives profile_id from device binding / browse cookie server-side; never trusts a client value (touchpoint #5) |
| T-03-11 | Elevation of Privilege | revoked device still served (missed SSE) | mitigate | per-request revoke guard (D3-07) returns 403 on revoked/unknown fingerprint independent of SSE delivery |
| T-03-12 | Information Disclosure | cross-profile data leak via stale binding | mitigate | profile_mismatch 403 retained; orphaned device falls back to picker, never another profile (D3-05) |
| T-03-13 | Denial of Service | SSE dep holds pool for stream lifetime | mitigate | device check acquires+releases pool inside the async dep; generator reads only the asyncio.Queue (Pitfall 10/3) |
| T-03-14 | Information Disclosure | fingerprint value exposed in session payload or logs | mitigate | only device_id (non-secret) exposed; fingerprint never serialized or logged |
</threat_model>
