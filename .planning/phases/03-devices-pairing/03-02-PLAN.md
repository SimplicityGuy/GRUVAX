<objective>
The endpoint layer for pairing and device administration. Kiosk-facing endpoints (`POST /api/devices/pairing-codes`, `GET /api/devices/me`) issue the fingerprint cookie and report device state. Admin endpoints (`/api/admin/devices/*`) handle the atomic PIN-gated bind plus the full lifecycle CRUD (list grouped, rename, change-profile, unbind, revoke, reinstate, delete), and publish `device_revoked`/`device_reassigned` on the per-profile SSE channel (D3-06). The per-request revoke guard, session-binding override, and profile soft-delete detach live in plan 03-03 (disjoint files).

Purpose: DEV-03 (4-digit code flow A: generate -> poll -> bind -> paired) and DEV-02 (admin binding + lifecycle CRUD + reassign SSE), with brute-force resistance (security touchpoint #4) and admin gating (touchpoint #8).
Output: 2 new routers + _BIND_RATE + router registration + SSE publish on mutations.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| kiosk → /api/devices/* | unauthenticated kiosk requests; server-authoritative code + cookie |
| admin browser → /api/admin/devices/* | PIN-session-gated mutations |
| admin → /api/admin/devices/bind | brute-force target (10k code keyspace) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Elevation of Privilege | bind code brute-force | mitigate | 5-min TTL × 10k keyspace × consumed_at one-shot × _BIND_RATE 10/5min × require_admin PIN-gate (touchpoint #4) |
| T-03-06 | Tampering | concurrent bind race on one code | mitigate | single atomic UPDATE ... WHERE consumed_at IS NULL AND expires_at > NOW() RETURNING fingerprint (PostgreSQL row lock; first wins) |
| T-03-07 | Spoofing | unauthenticated admin mutation | mitigate | every /api/admin/devices/* handler has Depends(require_admin) (touchpoint #8) |
| T-03-08 | Information Disclosure | fingerprint leaked in list/bind response or logs | mitigate | fingerprint never selected into or returned from any response (device_id only, non-secret per D3-05); never logged (Pitfall 7) |
| T-03-09 | Spoofing | client-supplied profile_id trusted on bind | accept | bind sets profile_id only via PIN-gated admin input; non-admin per-profile resolution derives profile_id from binding in 03-03 (touchpoint #5) |
</threat_model>
