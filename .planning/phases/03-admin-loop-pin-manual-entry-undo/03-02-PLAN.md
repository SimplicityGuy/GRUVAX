<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/admin/login | Untrusted PIN attempts cross from any LAN device |
| browser → /api/admin/* mutations | Forged cross-site/cross-device POSTs could mutate state |
| session cookie ↔ server | Session identity must be server-revocable, not just signed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Spoofing | POST /api/admin/login | mitigate | passlib Argon2id verify (memory-hard) + slowapi 5/5min per-IP rate limit → 429 (ASVS V2) |
| T-03-05 | Tampering | mutating /api/admin/* | mitigate | CSRF double-submit: X-CSRF-Token header must equal non-HttpOnly gruvax_csrf cookie; checked in require_admin for POST/PUT/PATCH/DELETE (Pitfall 13) |
| T-03-06 | Information Disclosure | login logging | mitigate | PIN never logged — log "pin_attempt=redacted" (Pitfall 12) |
| T-03-07 | Elevation of Privilege | stale/lost-device session | mitigate | Server-side admin_sessions rows + idle TTL (10min) + hard cap (30min) + Lock + logout/Change-PIN revocation (ASVS V3, Pitfall 23) |
| T-03-08 | Spoofing | session fixation | mitigate | New session UUID per login; Change-PIN revokes all other sessions |
| T-03-09 | Tampering | session cookie forgery | mitigate | gruvax_session signed via itsdangerous; HttpOnly + SameSite=Strict; invalid signature → 401 |
| T-03-SC | Tampering | (covered in plan 01) | accept | No new package installs beyond plan 01's [OK] passlib/slowapi |
</threat_model>
