<objective>
Deliver the boundary-editing vertical slice: an authenticated admin opens the cubes grid, taps a cube, and edits its first/last `(label, catalog#)` through a two-step dependent autocomplete fed exclusively by `gruvax.v_collection`. Saves are validated by the shared POS-01 comparator (no `first > last` slips through) and against the collection (phantom values are blocked with trigram near-misses + explicit override). An inline "Suggest midpoint" proposes a real owned record from collection-index space. This slice STOPS at building the `pendingChangeSet` — the diff preview + commit + history land in plan 05 (the validate endpoint here is the dry-run that powers the preview).

Output: `src/gruvax/api/admin/cubes.py` (read + validate + suggest, no writes yet), new admin queries, and the admin cubes grid + per-cube editor + autocomplete + A–Z rail + admin FillBar.
</objective>
