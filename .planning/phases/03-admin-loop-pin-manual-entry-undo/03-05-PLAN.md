<objective>
Close the admin loop: turn the `pendingChangeSet` accumulated by the editor (plan 04) into a reviewed, atomic, reversible commit. The admin sees a diff preview (mini-grid + per-cube before/after + record-movement counts), commits as one `change_set_id` written to `boundary_history`, and can revert any change-set by one tap — with conflict-aware skip+report so a newer edit is never silently clobbered. The boundary cache is invalidated and reloaded in-process after each commit so `/api/locate` and the kiosk reflect the new boundaries on next load.

Purpose: Satisfies ROADMAP criterion 3 (diff preview + boundary_history + change_set undo) — ADMN-07 + ADMN-09. This is the keystone that makes boundaries a maintained, recoverable artifact rather than a fixture, and the undo path Phase 6 wizards will reuse.

Output: `POST /api/admin/cubes/bulk` (atomic + idempotent), `src/gruvax/api/admin/history.py` (list + conflict-aware revert), and the frontend DiffPreviewSheet + HistoryView, wired into the editor's "PREVIEW CHANGES" → "COMMIT CHANGE SET" flow.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser → /api/admin/cubes/bulk | Privileged mutation of the live boundary table |
| admin browser → /api/admin/history/{id}/revert | Privileged state-restoring mutation |
| change-set history ↔ live boundaries | History is the audit + undo source of truth; must stay consistent with cube_boundaries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-18 | Spoofing/EoP | bulk + revert endpoints | mitigate | require_admin (session + CSRF) on both mutating routes — ASVS V4 |
| T-03-19 | Tampering | partial commit on flaky LAN | mitigate | Single DB transaction (all-or-nothing) + Idempotency-Key replay returns cached response, no double-write (Pitfall 7/11) |
| T-03-20 | Repudiation | boundary mutations | mitigate | Append-only boundary_history with prev/new values, source, changed_at per change-set (ADMN-09 audit trail) |
| T-03-21 | Tampering | revert clobbering newer edit | mitigate | has_newer_changes conflict check before writing inverse; skip+report, never silent clobber (D-12, Pitfall D) |
| T-03-22 | Denial of Service | stale boundary cache after failed commit | mitigate | invalidate()+load() only AFTER successful commit (Pitfall A) — a failed transaction never empties the live cache |
| T-03-23 | Denial of Service | idempotency_keys unbounded growth | mitigate | 24h cleanup DELETE on each bulk (indexed on created_at, Pitfall E) |
| T-03-24 | Tampering | SQL in bulk/revert | mitigate | All SQL %s placeholders, zero f-string interpolation (carried convention) |
</threat_model>
