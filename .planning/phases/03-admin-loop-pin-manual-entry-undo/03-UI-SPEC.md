# Phase 3 — UI Design Contract

> Visual and interaction contract for the admin loop: PIN gate, boundary editor,
> diff preview, change-set undo, fill-level indicator, and cube-contents panel.
> Generated by gsd-ui-researcher from upstream artifacts; all decisions are
> pre-populated from locked context (CONTEXT.md D-01..D-18) and the Nordic Grid
> design language (`design/gruvax-design-language.md`).

## Spacing Scale

Source: `design/gruvax-design-tokens.css` — `--gruvax-space-*` scale. All values
are multiples of 4px. Use token names in CSS; never write raw pixel values.

| Token               | Value  | Usage in this phase                                      |
|---------------------|--------|----------------------------------------------------------|
| `--gruvax-space-1`  | 4px    | Icon-to-label gaps, keypad key inner padding             |
| `--gruvax-space-2`  | 8px    | Compact form field gap, PIN dot spacing                  |
| `--gruvax-space-3`  | 12px   | Between form label and input, list item metadata gap     |
| `--gruvax-space-4`  | 16px   | Default field padding, section-within-card gap           |
| `--gruvax-space-5`  | 24px   | Card padding, panel section gap                          |
| `--gruvax-space-6`  | 32px   | Between major layout sections                            |
| `--gruvax-space-7`  | 48px   | Page-level vertical breathing room                       |
| `--gruvax-space-8`  | 64px   | Not used in admin (mobile viewport constraint)           |

**Phase 3 exceptions:**

- **Keypad tap targets:** Each numeric key must be at minimum 44×44pt (WCAG 2.5.5
  and D-17 / Pitfall 4). On kiosk at 80pt cells target 64×64px. On mobile minimum
  44×44px enforced via `min-height`/`min-width`.
- **A–Z jump rail:** Rail buttons are 32px wide × 44px tall — tall enough for touch,
  narrow to not crowd the list. Spacing between rail items: `--gruvax-space-1` (4px).
- **Session countdown pill:** Positioned fixed in the top-right corner of the admin
  chrome with `--gruvax-space-4` (16px) inset on both axes.

## Typography

All roles pull from the existing GRUVAX three-font system. Only roles relevant to
Phase 3 new surfaces are listed; inherited roles (search box, result rows, cube
address overlay) are unchanged from Phases 1–2.

### Admin surfaces — new roles

Phase 3 admin surfaces use exactly **4 font sizes**: 36px, 16px, 14px, 11px.
The 36px display size carries both page-level section headings (weight 900) and
keypad digits (weight 900) — they are differentiated by context, not by size.
All sub-headings, form labels, and body text collapse onto 16px and are
differentiated by font family and weight, not by an extra size step. This keeps
the type scale within the 4-size budget while preserving a clear
display → body → helper → micro hierarchy.

| Role                     | Font             | Size token                     | Weight | Line-height token              | Source              |
|--------------------------|------------------|-------------------------------|--------|-------------------------------|---------------------|
| Admin section heading    | Barlow Condensed | `--gruvax-text-display-lg` 36px | 900  | `--gruvax-leading-tight` 1.1  | design-language.md  |
| Keypad digit labels      | Barlow Condensed | `--gruvax-text-display-lg` 36px | 900  | `--gruvax-leading-tight` 1.1  | design-language.md  |
| Admin card sub-heading   | Barlow Condensed | `--gruvax-text-display-sm` 16px | 700  | `--gruvax-leading-tight` 1.1  | design-language.md  |
| Form label (ALL CAPS)    | Barlow Condensed | `--gruvax-text-display-sm` 16px | 700  | `--gruvax-leading-tight` 1.1  | design-language.md  |
| Diff before/after label  | Barlow Condensed | `--gruvax-text-display-sm` 16px | 700  | `--gruvax-leading-tight` 1.1  | design-language.md  |
| Form body / instructions | Space Grotesk    | `--gruvax-text-body` 16px       | 400  | `--gruvax-leading-normal` 1.5 | design-language.md  |
| Catalog number input     | DM Mono          | `--gruvax-text-mono-lg` 16px    | 500  | `--gruvax-leading-normal` 1.5 | design-language.md  |

| Form helper / hint       | Space Grotesk    | `--gruvax-text-body-sm` 14px    | 400  | `--gruvax-leading-normal` 1.5 | design-language.md  |
| Validation error message | Space Grotesk    | `--gruvax-text-body-sm` 14px    | 400  | `--gruvax-leading-normal` 1.5 | design-language.md  |
| Catalog number displayed | DM Mono          | `--gruvax-text-mono` 14px       | 400  | `--gruvax-leading-normal` 1.5 | design-language.md  |

| Diff boundary values     | DM Mono          | `--gruvax-text-mono` 14px       | 400  | `--gruvax-leading-normal` 1.5 | design-language.md  |
| Change-set timestamp     | DM Mono          | `--gruvax-text-mono-sm` 11px    | 400  | `--gruvax-leading-normal` 1.5 | design-language.md  |
| Fill % micro-label       | DM Mono          | `--gruvax-text-mono-sm` 11px    | 400  | `--gruvax-leading-tight` 1.1  | design-language.md  |
| Midpoint caption         | Space Grotesk    | `--gruvax-text-label` 11px      | 400  | `--gruvax-leading-tight` 1.1  | design-language.md  |

**Declared size set (4 total):** 36px, 16px, 14px, 11px.

**Weights in use (2 per family):** Barlow Condensed 700 + 900; Space Grotesk 400 +
500; DM Mono 400 + 500.

**Rules:**
- ALL CAPS labels (section headings, form field labels, keypad labels, button labels)
  use Barlow Condensed 700/900, `letter-spacing: var(--gruvax-tracking-label)` (0.14em).
- Section headings and keypad digits share the 36px size; they are distinguished by
  context (page heading vs. keypad cell), not by a separate size token.
- Sub-headings and form labels are 16px Barlow Condensed 700; body is 16px Space
  Grotesk 400 — distinguished by font family + weight, not by a separate size.
- Instructions and helper text are sentence case, Space Grotesk 400.
- Catalog numbers, bin positions, record counts, and change-set IDs always use DM Mono.
- Never use Barlow Condensed for body copy or multi-line instructions.

## Color

All colors consumed via CSS tokens. No hex values in component code.

### 60/30/10 split

| Role             | Token                       | Hex      | Usage                                                      | Source             |
|------------------|-----------------------------|----------|------------------------------------------------------------|--------------------|
| Dominant (60%)   | `--gruvax-white`            | #FFFFFF  | Page background, modal overlay ground, all admin surfaces  | design-language.md |
| Secondary (30%)  | `--gruvax-off-white`        | #F7F9FC  | Cards, form fields, panel backgrounds, list items          | design-language.md |
| Accent (10%)     | `--gruvax-blue`             | #0051A2  | See reserved list below                                    | design-language.md |
| Destructive      | `--gruvax-error`            | #C0392B  | Revert confirmations and destructive-action confirm only   | design-language.md |

### Accent (`--gruvax-blue`) reserved for

The 10% accent is used **only** on these specific elements — never as a generic
interactive color for every clickable thing:

1. Admin chrome top bar background (Reversed variant — blue ground, white text)
2. Primary CTA buttons ("SAVE BOUNDARIES", "COMMIT CHANGE SET", "CONFIRM REVERT")
3. The admin mini-grid diff: changed-cube highlight ring (2px `--gruvax-blue` border)
4. Form field focus ring (outline: 2px solid `--gruvax-blue`, offset 2px)
5. "SUGGEST MIDPOINT" inline action button text + underline
6. The A–Z jump rail active letter indicator

### Additional semantic tokens in use

| Situation                        | Token                         | Usage                                         |
|----------------------------------|-------------------------------|-----------------------------------------------|
| Fill level: low / normal         | `--gruvax-blue-light`         | Fill bar in cubes with < 80% fill             |
| Fill level: near-full (80–100%)  | `--gruvax-yellow`             | Fill bar highlight when approaching capacity  |
| Fill level: overstuffed (>100%)  | `--gruvax-error`              | Fill bar + cube ring tint                     |
| Phantom / near-miss warning      | `--gruvax-warning` (#E6A800)  | Warning icon, phantom label                   |
| Session countdown: normal        | `--gruvax-text-muted`         | Timer pill text                               |
| Session countdown: last 60 s     | `--gruvax-warning`            | Timer pill text transitions to warning color  |
| Successful save / commit         | `--gruvax-success` (#1A7A4A)  | Inline success indicator (1.5 s then fades)   |
| PIN overlay background           | `--gruvax-blue-darker` 95% opacity | Modal scrim over full viewport            |
| Admin chrome header background   | `--gruvax-blue`               | Top bar (Reversed variant)                    |
| Admin chrome header text/icons   | `--gruvax-white`              | On blue header                                |
| Cube state: admin selected/focus | `--gruvax-cell-selected`      | `#003D7A` — cube the editor is acting on      |

**Accessibility constraint (from design-language.md):**
- Blue on white: 7.2:1 (AAA — safe for body text).
- Yellow on blue / blue on yellow: 3.1:1 — large text (18px+) or decoration only.
  The keypad digit labels (36px Barlow Condensed 900) and section headings (36px) on
  the blue admin header bar are the only yellow-on-blue uses. Never use yellow-on-blue
  for 16px or smaller text.
- Error red on white: verify meets 4.5:1 in implementation.

| Mobile     | 480px     | Single-column; full-width cards; bottom sheets |
| Tablet     | 768px     | Two-column editor + mini-grid side-by-side     |

### State machine: PIN entry

```
[unauthenticated] → tap PIN digit × 4 → [4 digits entered] → POST /api/admin/login
  → success: session cookie set → [authenticated] → render admin chrome
  → failure (wrong PIN): shake animation + dot flash → [unauthenticated] (count++)
  → 429 rate-limited: lock message + retry countdown → [rate-limited] → [unauthenticated after delay]
```

- Each digit entry: visual feedback within `var(--gruvax-duration-fast)` (150ms).
- Wrong PIN shake: 300ms, 3 cycles ±6px translateX.
- After 4 digits entered, submit fires automatically — no confirm key.

### State machine: two-step autocomplete (D-06)

```
[empty] → type in label field OR tap label in list → [label selected]
  → catalog# field enables → type/tap catalog# → [catalog# selected]
  → save button enables
→ change label: catalog# field clears and disables
```

- Label field disabled until label selected on kiosk (tap-to-pick only, D-17).
- Catalog# field: `disabled` + `--gruvax-text-faint` text color until label chosen.
- On mobile, label field is a combobox with live filter; on kiosk, a scrollable full-width list.

### State machine: phantom (D-07)

```
[catalog# entered] → POST /api/admin/cubes/validate (dry-run)
  → match found: normal form state
  → no match (phantom): warning state → near-miss chips render
    → tap chip: fills catalog# field, re-validates → exits phantom state
    → tap "USE ANYWAY": confirm dialog → on confirm: `force: true` flag set
```

- Phantom state does not block navigation but does block the "SAVE BOUNDARIES" button.
- "USE ANYWAY" is secondary, destructive-styled — not the primary path.

### Session countdown behavior (D-04)

- Timer starts on login; counts down from `idle_ttl_seconds` (default 600 = 10 min).
- Resets to full TTL on every server round-trip (sliding window).
- At T-60s: countdown text color transitions to `--gruvax-warning` over 1s.
- At T-0: session expires → PIN overlay re-appears with message "Session expired. Enter PIN to continue editing." — `pendingChangeSet` is preserved in localStorage.
- Hard cap (default 30 min): at T-5min: inline banner appears. At T-0: force logout regardless of activity.

### Fill-level animation

- Fill bars animate from 0 to their computed width on page load: 400ms `var(--gruvax-ease-decelerate)`.
- On boundary commit, affected cube fill bars re-animate to new value: 400ms `var(--gruvax-ease-standard)`.
