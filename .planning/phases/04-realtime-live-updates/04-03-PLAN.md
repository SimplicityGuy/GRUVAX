<objective>
Add the two admin-side realtime behaviors: (1) the `admin_editing` heartbeat — a session + CSRF-gated `POST /api/admin/editing` that fans out an `admin_editing` event so the kiosk can shimmer the affected range while the owner is mid-edit (RTM-04 backend); and (2) optimistic admin edits with rollback — the admin commit applies optimistically to the owner's own device, and on server rejection it reverts, shows a plain-language toast, and keeps the attempted values for retry (RTM-03). Both are owner-device / admin-surface only; the public kiosk never sees uncommitted state.

Purpose: Satisfies RTM-04 (the heartbeat that drives the kiosk shimmer — kiosk rendering of that shimmer is Plan 04) and RTM-03 (optimistic UI with rollback). Output: New `editing.py` sub-router + registration, the heartbeat client call + debounce, the `putCubeBoundary` client fn, the optimistic `useMutation` at the real commit site, the `RollbackToast` component + admin.css, and integration + frontend tests.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin device → POST /api/admin/editing | Untrusted client write; must be session + CSRF gated like every admin write |
| admin device → admin commit | Owner-device-local optimistic state that must never leak to the public kiosk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | Elevation of Privilege | POST /api/admin/editing opened as an unauthenticated write | mitigate | Endpoint depends on require_admin (session cookie + CSRF double-submit), identical gating to existing admin writes. test_editing_requires_auth asserts 401/403 with no session. Never an open write. |
| T-04-09 | Denial of Service | Admin spamming admin_editing to flood the bus | mitigate | Session+CSRF gate limits callers to the authenticated owner; client debounces to ~300ms; the bus drops on QueueFull (put_nowait) so a flood never blocks publishers or other subscribers (T-04-03 from Plan 01). |
| T-04-10 | Information Disclosure | Optimistic admin state visible on the public kiosk (D-08 violation) | mitigate | Optimistic setQueryData targets admin-only keys (['admin','cubes'], ['admin','cube-boundary',...]); the kiosk reads ['cubes']/['cube-contents'] and only updates on committed boundary_changed SSE. onSettled never invalidates kiosk keys — asserted by the DiffPreviewSheet test. |
| T-04-11 | Input Validation | admin_editing body injection | mitigate | Body validated by Pydantic EditingPayload (typed cube_ids ints + editing bool); model_dump() emits only validated fields onto the bus; no raw client string echoed. |
| T-04-SC | Tampering | npm/pip installs | mitigate | No new packages (RESEARCH § Package Legitimacy Audit). Lucide already in use. No install task → no checkpoint required. |
</threat_model>
