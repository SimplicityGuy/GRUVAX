<objective>
Fix the UAT Test 1 blocker (09-HUMAN-UAT.md): the Phase 9 offline banner and degraded read-only
mode render whenever `sseConnected === false`, which INCLUDES the never-connected state (initial
bootstrap, or an SSE connection rejected before it ever opens — e.g., 403 device_unknown for an
unpaired/stale-device kiosk). Result: the banner shows "Can't reach GRUVAX — trying to reconnect…"
on initial load and never clears, bricking the kiosk even though the server is reachable. This
masks a never-connected/terminal state — exactly the risk the verifier flagged.

Fix: introduce `everConnected` (true after the first successful SSE `onopen`) and make the offline
banner + degraded-mode lockouts represent "offline-confirmed" = `!sseConnected && everConnected`,
i.e. a connection that was established and then LOST — never a connection that never opened.
Preserves SC1 (connect → stop server → banner) and SC3 (reconnect → banner clears).

This is a frontend-only fix. Do NOT change backend device-guard semantics (deps.py / D3-07) —
routing device_unknown to pairing is a separate pairing-phase concern, out of scope here.
</objective>
