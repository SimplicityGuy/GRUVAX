<objective>
Replace the stdlib `JsonFormatter` + `LogRingHandler` wiring with a `structlog`-based
configuration that preserves the in-memory log-ring contract `/admin/diagnostics` reads
(D-01 structlog migration), and lock in the env-driven log level (D-02 — already plumbed
via `LOG_LEVEL`, this plan verifies + tests it).

survive byte-compatible is `GET /api/admin/diagnostics` tailing recent `gruvax`-logger
messages as `{ts, level, logger, msg}` dicts, with third-party records excluded (a security
control — third-party records may carry a DSN/secret).

Purpose: Modernize logging plumbing onto structlog (sibling-consistency with discogsography)
with zero product-behavior change.
Output: New `configure_logging()` in `logging_config.py`, lifespan rewired in `app.py`,
`structlog`+`orjson` deps added, Wave-0 regression assertions added to the diagnostics test,
and a new unit test for the env-driven log level.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| third-party logger → log ring buffer | Records from psycopg/uvicorn/etc. may stringify a DSN or secret; they must NOT reach the in-memory ring that `/admin/diagnostics` exposes |
| log ring buffer → admin UI | `/admin/diagnostics` (PIN-gated) returns ring contents; the ring must contain only `gruvax`-scoped records |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-9-IL | Information Disclosure | `LogRingHandler` scoping in `configure_logging()` | mitigate | Attach the ring handler ONLY to `logging.getLogger("gruvax")`, never root (preserves the pre-Phase-9 control). Regression-asserted in Task 3 (third-party logger absent from `recent_logs`). |
| T-9-TAMPER | Tampering | structlog record passed to `LogRingHandler.emit()` | mitigate | emit() only READS `record.msg`; never mutates the record (ProcessorFormatter needs it intact for stdout). Verified by ruff/mypy + the unchanged stdout JSON shape. |
| T-9-SHAPE | Repudiation | ring entry shape consumed by diagnostics | accept→mitigate | Shape change would silently break the admin diagnostics view; mitigated by the Task 3 key/type assertion on every `recent_logs` entry. |
</threat_model>
