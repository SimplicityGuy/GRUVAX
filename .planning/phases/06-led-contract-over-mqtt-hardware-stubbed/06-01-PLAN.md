```python
@dataclass(frozen=True)
class CubeRef: unit_id: int; row: int; col: int
@dataclass(frozen=True)
```

NOTE: sub_cube_interval JSON has NO `cube` field — frontend derives cube from primary_cube/label_span context. For sub-topic publish, use primary_cube's unit/row/col.

<source_of_truth>
ARCHITECTURE §"MQTT Topic Design" is the LOCKED contract:
- Topics: illuminate/{unit_id}/{row}/{col}, span/{change_id}, sub/{unit_id}/{row}/{col}, all/off, diagnostic, state/{unit_id}/{row}/{col} (RETAINED), status/{unit_id}
- QoS: illuminate/span/sub = QoS 0 non-retained; all/off + diagnostic = QoS 1 non-retained; state/* = QoS 1 RETAINED
- Payload schema names: gruvax.illuminate.v1, gruvax.sub_interval.v1, span uses gruvax.span.v1
- sub interval is normalized 0..1 within cube pixel range; firmware translates to pixel indices (LED-03 pixel_start/pixel_end semantics are the interval.start/interval.end)
NEVER retain command topics (illuminate/span/sub) — only state/* is retained (Pitfall: stale command replay).
</source_of_truth>
