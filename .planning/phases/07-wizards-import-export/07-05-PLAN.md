<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin upload → import UI → server | Untrusted file selected in the browser crosses to the import endpoint |
| export download → owner's disk | Boundary/settings YAML leaves the app as a shareable file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-XSS | Tampering | Import error cards / history badges / diff labels | mitigate | JSX `{}` interpolation only for label/catalog/change_set_id/source; no innerHTML (sketch-findings constraint) |
| T-07-PARTIAL-UI | Tampering | Import commit gating | mitigate | COMMIT IMPORT disabled until zero validation errors (D-11); server re-validates authoritatively (Plan 03) — the UI gate is convenience, not the guard |
| T-07-MISLEAD | Information Disclosure (UX) | movement-count diff | accept | Counts are approximate (records_after==records_before); UI labels every non-zero delta "(approx.)" so the owner is not misled (Pitfall 5) |
| T-07-PIN-LEAK-UI | Information Disclosure | settings export download | mitigate | Server export is allowlist-only (Plan 03, T-07-PIN-LEAK); the UI only triggers the GET — the PIN is never in the payload it downloads |
| T-07-SC | Tampering | npm installs | mitigate | No new npm packages (RESEARCH §No New Dependencies). No legitimacy checkpoint required. |
</threat_model>
