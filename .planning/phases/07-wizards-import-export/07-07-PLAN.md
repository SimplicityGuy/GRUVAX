<objective>
Close two backend gaps on the boundary-import path:

- **G2 (UAT test 3, major):** Import has NO validate-only/dry-run pass. `Import.tsx` calls the
  atomic COMMIT endpoint for its "validation pass", so a CLEAN file is WRITTEN on upload and the
  later COMMIT IMPORT click is an idempotency no-op (preview is AFTER the write). Add a
  `dry_run` mode to `POST /api/admin/import/boundaries` that runs the EXACT same parse +
  full-address-fill (D-09) + validation path but returns the diff + per-row errors WITHOUT any
  DB write. The preview thus matches the commit byte-for-byte because it is the same code path.

- **G3 (UAT test 4, SC4, major) — DECISION LOCKED:** In `import_boundaries`, SKIP phantom
  re-validation for any row that EQUALS the current committed cut point
  (`unit_id,row,col,first_label,first_catalog,is_empty`). New/changed rows still get full phantom
  + contiguity validation. This guarantees export → re-import = identity with the smallest change.
  (Do NOT implement the rejected alternatives: stable-record-id round-trip; reject-non-matching
  pairs on write.)

G4's backend side is already correct (the endpoint reads raw `request.body()` and the host
integration tests already send raw bytes) — the client mismatch is fixed in plan 07-08. This
plan ONLY touches the backend + tests so it stays parallel to the frontend plans.

identity, SC4).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /api/admin/import/boundaries | Untrusted file upload (raw body) crosses here; admin-gated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-0707-YAML-BOMB | Tampering / DoS | import_.py YAML parse | mitigate | `yaml.safe_load` ONLY (never `yaml.load`); 100 KB size cap enforced BEFORE parse in both dry_run and commit branches (keep the existing cap ahead of branching). |
| T-0707-DRYRUN-WRITE | Tampering | dry_run branch | mitigate | dry_run MUST perform NO INSERT/UPDATE/DELETE and MUST NOT invalidate cache or publish on the bus — assert via the round-trip test (preview shows zero diff yet a fresh export still matches; no change_set_id minted in preview). |
| T-0707-IDENTITY-BYPASS | Tampering (validation weakening) | G3 phantom skip | mitigate | Skip phantom ONLY for rows byte-equal to the current committed cut point; NEW/CHANGED rows keep full phantom + near-miss; contiguity always runs across all rows. test_phantom_row_rejected + test_contiguity_violation guard against over-broad skip. |
| T-0707-AUTH | Spoofing / EoP | import endpoint | mitigate | `require_admin` (session + double-submit CSRF) unchanged on the POST; dry_run adds no new auth surface (still a POST through require_admin). |
