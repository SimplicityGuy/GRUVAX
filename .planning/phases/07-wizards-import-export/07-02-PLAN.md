<objective>
Build the import/export transform layer: one internal `CutPointEntry` model and two parsers
(YAML, CSV) plus a YAML serializer, so import and export both speak a single cut-point shape
(D-08, D-10). The round-trip identity property (export → re-import → zero diff, SC4) lives
entirely in this layer and is provable without a DB.

Purpose: Centralizes all file-format handling in a new `src/gruvax/io/` package, isolated from
the HTTP/DB layer (cleanly unit-testable). Enforces `yaml.safe_load` (the YAML-bomb / arbitrary-tag
RCE guard) at the only place YAML is parsed.

Output: `src/gruvax/io/` package with `boundary_yaml.py` (model + parse + serialize) and
`boundary_csv.py` (flat parse). No HTTP, no DB — pure transforms.
</objective>

Internal cut-point model (RESEARCH §Concrete Schemas — Internal Cut-Point Model):
  @dataclass CutPointEntry:
    unit_id: int; row: int; col: int
    first_label: str | None; first_catalog: str | None
    is_empty: bool
    overrides: dict[str, float]   # label -> fraction in (0.0, 1.0]; empty dict when none

YAML schema (RESEARCH §Concrete Schemas — YAML Import/Export Schema):
  top-level: {version: "1", cubes: [ {unit_id,row,col,first_label,first_catalog,is_empty,overrides?}, ... ]}
  serialize with: yaml.dump(..., default_flow_style=False, allow_unicode=True, sort_keys=True)

CSV schema (RESEARCH §Concrete Schemas — CSV Import Schema):
  required headers: unit_id,row,col,first_label,first_catalog,is_empty
  is_empty truthy values: "true"/"1"/"yes" (case-insensitive); CSV carries NO overrides

segment_overrides table shape (VERIFIED from segments.py): columns (unit_id,row,col,label,fraction,updated_at);
  fraction CHECK 0 < fraction <= 1.0 — match this range in the model.
