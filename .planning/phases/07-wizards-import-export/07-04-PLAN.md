<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser localStorage → commit | Draft cuts persisted client-side cross into the validate/commit calls |
| wizard client → admin write endpoints | Authenticated admin's proposed cut set crosses to the server |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-STALE-DRAFT | Tampering | reshuffleDraft on resume | mitigate | Continue re-validates via POST /cubes/validate against current v_collection; stale records flagged with did-you-mean (D-06). Server phantom/contiguity check is authoritative — the client can lie. |
| T-07-DOUBLE-COMMIT | Tampering (dup data) | Wizard commit | mitigate | crypto.randomUUID() Idempotency-Key persisted in the draft before the network call; reused on retry; server dedup (Pitfall 4) |
| T-07-CSRF | Spoofing | adminClient mutating calls | mitigate | adminFetch auto-injects X-CSRF-Token read at call time (re-auth-safe, Pitfall 2) |
| T-07-XSS | Tampering | wizard/confirmation rendering of label/catalog/change_set_id | mitigate | JSX `{}` interpolation only (textContent semantics); never innerHTML (sketch-findings constraint) |
| T-07-SC | Tampering | npm/cargo installs | mitigate | No new npm packages (RESEARCH §No New Dependencies; crypto.randomUUID is a browser builtin). No legitimacy checkpoint required. |
</threat_model>
