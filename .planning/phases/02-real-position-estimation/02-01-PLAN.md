Swap the real §4.1 index-based estimator in behind the locked `LocateResult` contract so `/api/locate` answers "where exactly in the cube" — a populated `sub_cube_interval` with a calibrated confidence — while keeping the Phase 1 §4.8 cube-only path as the timeout/low-confidence fallback. NO_BOUNDARY_CONFIDENCE: float = 0.0  (in contract.py)

<!-- Existing §4.8 estimator — extend, do not rewrite. From src/gruvax/estimator/algorithm.py -->
def locate_cube_only(release_id: int, label: str, catalog_number: str, cache: BoundaryCache) -> LocateResult
  - covering = boundary rows where first_label.casefold() <= label.casefold() <= last_label.casefold()
    AND catalog_in_range(catalog_number, b.first_catalog, b.last_catalog)
  - sorted_span = sorted(covering, key=lambda c: (c.unit_id, c.row, c.col))
  - primary_cube = sorted_span[0]; returns confidence CUBE_ONLY_CONFIDENCE when covered, else NO_BOUNDARY_CONFIDENCE + primary_cube=None
