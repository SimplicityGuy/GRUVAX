## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| member browser (untrusted) → POST /api/invite-codes/{code}/redeem | PUBLIC endpoint accepting a secret (member PAT) — the elevated threat surface of this phase |
| member browser → GET /api/invite-codes/{code} | PUBLIC code-validation endpoint — must not become a code-enumeration oracle |
| invite_codes server → discogsography HTTP | outbound PAT validation; PAT must not be logged or leaked |
| invite_codes server → Postgres | encrypted-at-rest PAT store; atomic single-use consume |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Elevation of Privilege | invite code brute-force / enumeration | mitigate | UUID4 = 122 bits entropy (gen_random_uuid); per-IP rate limit 5/10min via _REDEEM_RATE on redeem; GET validate returns uniform 404 and never echoes the code. |
| T-07-06 | Tampering | replay / double-redeem | mitigate | Atomic `UPDATE ... WHERE consumed_at IS NULL AND expires_at > NOW() RETURNING profile_id`; second request matches no row → 404 (PostgreSQL row-lock first-wins). |
| T-07-07 | Information Disclosure | PAT exposure via API response | mitigate | Redeem returns only {status, profile_id}; PAT is never echoed; admin API exposes derived `has_token` only (Plan 01). |
| T-07-08 | Information Disclosure | PAT exposure via logs | mitigate | No error/detail/log string is constructed from body.pat; existing structlog log_redactor masks dscg_* substrings as defense-in-depth. |
| T-07-09 | Information Disclosure | PAT confidentiality at rest | mitigate | `encrypt_pat()` (Fernet/authenticated encryption) before the bytea write; the owner holding the DB cannot read the plaintext. |
| T-07-10 | Information Disclosure | oracle: expired vs used vs invalid | mitigate | All negative invite cases return a uniform 404 {type: invite_not_found}; no status-code or body discriminator (Pitfall 2). |
| T-07-11 | Denial of Service | pool exhaustion via concurrent redeems | mitigate | Pool-isolation discipline: `_run_test_sync` HTTP call runs with NO pool slot held (Pitfall 1); per-IP rate limit caps abuse. |
| T-07-12 | Spoofing | redeem onto a profile that already has a token | accept | D-10 intentionally overwrites/rotates the token; safe because only the owner can mint a single-use TTL-bounded invite for that profile. |
| T-07-LAN | Information Disclosure | PAT over plaintext HTTP on LAN | accept | L-05 locked: home-LAN-only, no public exposure; documented as a runbook note in the SUMMARY. TLS required only if ever exposed beyond the LAN. |
| T-07-SC | Tampering | npm/pip/cargo installs | mitigate | RESEARCH §Package Legitimacy Audit confirms NO new packages this phase; no install task exists. |
