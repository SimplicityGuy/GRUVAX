## Color

All via `var(--gruvax-*)` tokens. Never hardcode hex.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--gruvax-white` (`#FFFFFF`) | Redeem page background; admin sheet background |
| Secondary (30%) | `--gruvax-off-white` (`#F7F9FC`) | Redeem page card zone; invite-link display box; diagnostics card background (established: `.diag-profile-card`) |
| Accent (10%) | `--gruvax-yellow` (`#FFDA00`) | See reserved list below |
| Destructive | `--gruvax-error` (`#C0392B`) | None in this phase — no destructive actions |

**Accent (`--gruvax-yellow`) is reserved for:**
- The "N new records" indicator badge/count number (kiosk only — yellow-on-blue text or yellow pill, large text ≥ 18px)
- The focus ring on the PAT input on the redeem page (`outline: 2px solid var(--gruvax-yellow)` — consistent with existing search bar / kiosk focus pattern)
- The "COPY LINK" primary CTA button background (admin — yellow CTA follows the "active / changed / lit" rule)

**Accent is NOT used for:**
- Body copy or instructions anywhere
- The "N new records" indicator in the admin diagnostics card (blue/success palette, not yellow — yellow reserved for kiosk; admin uses `--gruvax-success` green or `--gruvax-text-primary` blue)
- Any text below 18px

**State colors for the "N new records" indicator:**
- Admin diagnostics card: `--gruvax-success` (`#1A7A4A`) — a new record arrival is a positive event; green dot + count in DM Mono
- Kiosk: yellow pill (`--gruvax-yellow` background, `--gruvax-blue-darker` text) — reinforces the LED visual language; large text (18px+) satisfies the 3.1:1 contrast constraint for yellow-on-blue

**Redeem page error states:**
- Expired code: `--gruvax-warning` (`#E6A800`)
- Used/invalid code: `--gruvax-error` (`#C0392B`)
- Validation in-progress: `--gruvax-text-muted` with `Loader2` spinner

## Surface Contracts

### Surface 1 — Redeem Page (`/redeem/:code`)

**Layout:** Full-page, single-column, centered card. Mobile-first (390px baseline). No `/admin` PIN gate — fully public route.

**Structure (top to bottom):**
1. GRUVAX icon mark (`gruvax-logo-icon.svg`, 48px, Standard variant on white ground)
2. Heading: "CONNECT [PROFILE DISPLAY_NAME]" — Barlow Condensed 900 24px `--gruvax-blue`, ALL CAPS (display_name uppercased in render)
3. Instruction paragraph: "To connect [display_name]'s collection, find your Discogs personal access token at discogs.com/settings/developers, then paste it below." — Space Grotesk 400 16px sentence case
4. Discogs link: "Open Discogs developer settings" — `--gruvax-blue`, underlined, opens `https://www.discogs.com/settings/developers` in a new tab (`target="_blank" rel="noopener noreferrer"`)
5. Field label: "PERSONAL ACCESS TOKEN" — Barlow Condensed 700 16px tracked
6. PAT input: `type="password"` + Eye/EyeOff toggle (lucide-react, 16px). Focus ring: 2px `--gruvax-yellow`. Placeholder: "Paste your token here". `autocomplete="off"`.
7. CTA button: "CONNECT COLLECTION" (Barlow Condensed 700 16px, `--gruvax-blue` background, `--gruvax-white` text, full-width, min-height 44px). While submitting: "CONNECTING…" + `Loader2` spinner.
8. Error zone (conditional, `role="alert"`): full-width, see Copywriting Contract.

**States:**

| State | What shows |
|-------|-----------|
| Loading (GET to validate code) | Spinner centered; heading and form hidden |
| Expired code | Error card (no form) — see copy below |
| Used code | Error card (no form) — see copy below |
| Invalid code | Error card (no form) — see copy below |
| Active (normal) | Full form as described above |
| Submitting | CTA disabled + "CONNECTING…" + spinner; input disabled |
| PAT rejected | Inline error under PAT input (form stays interactive) |
| Success (terminal) | Replace form with terminal success state — no close button, no spinner |

**Terminal success state:**
- Checkmark icon (`--gruvax-success` color, 32px)
- Heading: "CONNECTED" — Barlow Condensed 900 24px `--gruvax-success`
- Body: "Your collection is importing. You can close this page." — Space Grotesk 400 16px
- No further interaction. Auto-sync runs server-side per D-04.

**Mobile constraints:**
- Max width: 480px, centered with horizontal padding `--gruvax-space-5` (24px)
- On screens < 390px: horizontal padding reduces to `--gruvax-space-4` (16px)
- Viewport height: use `min-height: 100dvh` (dynamic viewport height for mobile browsers)
- Touch target: all interactive elements min 44px height
- No bottom sheet / no drawer — full page card only

### Surface 2 — Admin Invite Affordance (ProfileDrawer / ProfilesManager)

**Placement:** Inside `ProfileDrawer` for profiles with `status === 'connected'` or `status === 'pending'` AND `has_token === false` (pending), OR for already-connected profiles where owner wants to invite a member to deposit their own PAT. The affordance appears in the `drawerMode === 'view'` state, below the SYNC NOW / ROTATE PAT row.

**Structural additions to ProfileDrawer (existing bottom-sheet pattern):**

New section between existing action rows and DELETE PROFILE:

```
INVITE LINK                           (section label, Barlow Condensed 700 16px)
┌─────────────────────────────────────────┐
│ http://gruvax.lan/redeem/xxxxxxxx       │  ← DM Mono 14px, --gruvax-text-muted
│ Expires in 47:23                        │  ← Space Grotesk 14px, --gruvax-text-muted
└─────────────────────────────────────────┘
[ COPY LINK ]                             ← yellow CTA button
```

When no active invite exists:

```
INVITE LINK                           (section label)
[ GENERATE INVITE LINK ]              ← secondary button (--gruvax-blue outlined)
```

**Invite-present sub-states:**

| Sub-state | Display |
|-----------|---------|
| No active invite | "GENERATE INVITE LINK" outlined secondary button |
| Active invite (not yet copied) | Link box + TTL countdown + "COPY LINK" yellow CTA |
| Just copied (2s feedback) | "COPIED!" text replaces button label (Barlow Condensed 700 16px `--gruvax-success`), then reverts |
| TTL expired (countdown reaches 0:00) | Link box grays out (`--gruvax-text-faint`), replace CTA with "GENERATE NEW LINK" |

**TTL countdown:** Updates every second via `setInterval`. Format: `MM:SS`. When under 5 minutes, color shifts from `--gruvax-text-muted` to `--gruvax-warning`. When under 1 minute, `--gruvax-error`.

**One active invite rule (D-09):** Generating a new invite immediately supersedes the prior one server-side. The UI shows only one invite block — no history.

**Owner never sees raw PAT:** The owner sees only `has_token: bool` in profile responses. No PAT display anywhere in admin UI.

**Copy mechanism:** `navigator.clipboard.writeText(url)`. On failure (clipboard permission denied): show inline error: "Could not copy. Tap the link to copy manually." Link becomes selectable text.

### Surface 3 — "N New Records" Indicator

**Delivery:** `collection_changed` SSE payload, extended with `new_record_count: number` and `is_initial_import: boolean`. Persists until next sync (D-08) — derived from stored count on the profile, not transient toast state.

**Admin diagnostics card (existing `ProfileDiagnosticsCard`):**

New row added between ITEMS and LAST ERROR:

```
NEW RECORDS    42                       ← label (Barlow Condensed 700 16px) + count (DM Mono 14px --gruvax-success)
```

- Row label: "NEW RECORDS" — `.diag-row-label` (Barlow Condensed 700 16px, established class)
- Count: DM Mono 14px, `--gruvax-success` for non-zero values; `--gruvax-text-muted` for zero
- Initial import variant: "IMPORTED" label instead of "NEW RECORDS", count in `--gruvax-text-primary`
- Zero state: show "—" in `--gruvax-text-muted` (consistent with ITEMS row zero pattern)
- No yellow used on admin cards (established rule from existing `ProfileDiagnosticsCard` comment: "No yellow on this card")

**Kiosk indicator (new, KioskView.tsx):**

Persistent pill in the kiosk header area, visible when `new_record_count > 0`.

```
┌──────────────────────┐
│  42 NEW RECORDS  →   │   ← yellow pill
└──────────────────────┘
```

- Background: `--gruvax-yellow`; text: `--gruvax-blue-darker`; border-radius: `--gruvax-radius-pill`
- Font: Barlow Condensed 700 18px (large text — satisfies 3.1:1 contrast for yellow-on-blue)
- Count in DM Mono 18px weight 500 inline (same color)
- Padding: `--gruvax-space-2` vertical, `--gruvax-space-4` horizontal
- Position: below search box, above results list — full width on mobile, auto-width on larger screens
- Initial import variant: "IMPORTED 2,847 RECORDS" (same pill, Barlow Condensed 700 18px)
- Dismisses automatically on next `collection_changed` SSE event (D-08)
- Transition in: `opacity 0 → 1` over `--gruvax-duration-base` (250ms), `--gruvax-ease-decelerate`
- Transition out: `opacity 1 → 0` over `--gruvax-duration-fast` (150ms)
- No manual dismiss button (D-08: persists until next sync, not user-dismissible)

## Interaction Contracts

### Redeem Page Load Sequence

1. Mount → immediately call `GET /api/invite-codes/:code` (or equivalent public endpoint)
2. While loading: full-page centered `Loader2` spinner, `--gruvax-blue`, 32px
3. On 404/expired/used: replace spinner with error card (no form rendered)
4. On 200: render form with profile `display_name` hydrated into heading

### Invite Generation Flow

1. Tap "GENERATE INVITE LINK" → POST to generate endpoint
2. While in-flight: button shows `Loader2` + "GENERATING…"
3. On success: transition to invite-present state; start TTL countdown interval
4. On error: inline error below section label, button re-enabled

### Copy Flow

1. Tap "COPY LINK" → `navigator.clipboard.writeText(fullUrl)`
2. On success: button label → "COPIED!" for 2000ms, then reverts to "COPY LINK"
3. On failure: inline error message; URL box becomes user-selectable text

### SSE Wiring for Kiosk (API-04)

The kiosk `KioskView.tsx` currently ignores `collection_changed`. This phase wires it:

1. Existing `EventSource` in `KioskView.tsx` handles `collection_changed` event
2. On event: parse `{ profile_id, new_record_count, is_initial_import }` from payload
3. Verify `profile_id` matches `boundProfileId` (per-profile fan-out, established pattern)
4. Update local state: `newRecordCount` (number), `isInitialImport` (boolean)
5. Show kiosk pill when `newRecordCount > 0`
6. On next `collection_changed` event: clear previous count, render new count

### Accessibility

- Redeem page: `role="main"`, `<h1>` for heading, `aria-label` on PAT toggle, `role="alert"` on error zone
- Success terminal: `role="status"` on success message
- Admin invite section: `aria-live="polite"` on TTL countdown; `aria-label="Copy invite link"` on copy button
- Kiosk pill: `role="status"`, `aria-live="polite"`, `aria-label="[N] new records since last sync"` (or "Imported [N] records" for initial import)
- Focus trap: redeem page is a standalone page — no focus trap needed. Admin invite section uses existing bottom-sheet focus trap.
- All interactive elements: focus ring via `outline: 2px solid var(--gruvax-blue); outline-offset: 2px` (established `.profile-btn-*` and `.diag-btn-*` pattern). Exception: PAT input on redeem page uses `outline: 2px solid var(--gruvax-yellow)` — matches kiosk search bar pattern.

## Responsive Breakpoints

**Redeem page (Surface 1):**
- Mobile (< 640px): full-width card, `padding: 0 var(--gruvax-space-4)`, stacked layout
- Tablet+ (≥ 640px): centered card `max-width: 480px`, `margin: 0 auto`
- Kiosk (1024px+): redeem page is never shown on the kiosk — it is opened on the member's own device

**Admin surfaces (Surfaces 2 + 3):**
- Mobile-first, consistent with existing admin CSS (`admin.css` uses 390px baseline)
- Invite section: full-width within the existing bottom sheet (no change to sheet width)
- Admin diagnostics card: new row follows established `.diag-status-row` responsive flex pattern

**Kiosk indicator (Surface 3):**
- Full-width below search box on the 7" kiosk display (800×480 or similar)
- Auto-dismisses from layout when `newRecordCount === 0` (no reserved space)
