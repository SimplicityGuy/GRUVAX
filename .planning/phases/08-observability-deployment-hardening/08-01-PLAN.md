<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LAN client → /api/version | Public, unauthenticated endpoint on a LAN-only box |
| Background task → gruvax.v_collection | Internal DB read of discogsography sync timestamp (Pitfall 5: view only) |
| Log records → in-memory log ring buffer | Any module's log lines accumulate in memory, later read by admin diagnostics |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | GET /api/version | mitigate | Return only `git_sha`/`build_timestamp`/`environment`; never read settings/env/DSN/PIN. Integration test asserts no secret keys in body. |
| T-08-02 | Information Disclosure | log ring buffer | mitigate | JsonFormatter logs `record.getMessage()` only; codebase logs counts/timing, never raw query text or PIN. Buffer is read solely via the admin-gated diagnostics endpoint (Plan 04). |
| T-08-03 | Denial of Service | sync-age refresh task | accept | 60 s cadence + single-row aggregate over ≤3K rows; trivial load. Failures caught, logged, never crash startup. |
| T-08-04 | Tampering | npm/pip/cargo installs | mitigate | No new packages this plan (RESEARCH Package Legitimacy Audit: none); stdlib + installed deps only. |
</threat_model>
