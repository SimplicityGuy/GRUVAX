<objective>
Close DATA-01: make every admin boundary write profile-scoped and fan out its SSE
events to the resolved profile's bus only.

Currently `write_boundary` (queries.py:641) omits `profile_id` from its WHERE
clause. On a multi-profile DB (PK `(profile_id, unit_id, row, col)`, migration 0010)
an edit can clobber the wrong — or every — profile's row at the same physical
position. All six write call sites also publish `boundary_changed` / `admin_editing`
on the default `app.state.event_bus`, leaking admin SSE events across profiles.

Purpose: eliminate the cross-profile corruption hole and SSE leakage that block
safe multi-profile boundary editing.
Output: scoped `write_boundary` (returns affected-row count), one shared
resolve-and-bus dependency, all 6 call sites scoped + retargeted, loud 404 on
0-row writes, transactional bulk abort.
</objective>
