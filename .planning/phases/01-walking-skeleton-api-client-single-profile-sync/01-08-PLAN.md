<objective>
Close the WARNING Gap #2 from 01-VERIFICATION.md by rewriting `tests/integration/test_migrate_0009.py::_alembic()` to drive Alembic via `subprocess.run()` instead of programmatic `alembic.command.upgrade(...)`. The subprocess has its own Python interpreter and no parent event loop, so `env.py`'s internal `asyncio.run(run_async_migrations())` succeeds — exactly the way `just migrate-roundtrip` already works in CI.

Purpose: Currently all 7 tests in `test_migrate_0009.py` ERROR with `RuntimeError: asyncio.run() cannot be called from a running event loop` because the async test (running under pytest-asyncio's event loop) calls sync `alembic.command.upgrade()` which internally `asyncio.run()`s env.py's async migration runner — and Python forbids nested `asyncio.run()` from within an existing loop. The standalone `just migrate-roundtrip` CLI recipe works because it spawns a fresh `alembic` process; the in-pytest version provides zero evidence today. After this fix the SC-3 round-trip cleanness is provable from BOTH paths (CI shell recipe AND in-pytest schema-verification tests).

Output:
- `tests/integration/test_migrate_0009.py::_alembic()` rewritten to invoke `subprocess.run(["uv", "run", "alembic", action, target], ...)` with capture_output + reasonable timeout + manual returncode check (so failure surfaces include alembic stdout/stderr in the pytest report). The async signature is kept (tests still `await _alembic(...)`) by wrapping `subprocess.run` in `asyncio.to_thread(...)` so the test event loop remains unblocked during the subprocess wait.

Replace the body of `_alembic()` in `tests/integration/test_migrate_0009.py` (lines 56-76) with a subprocess-driven implementation. Keep the `async def _alembic(action: str, target: str) -> None:` signature so all three existing call sites (`fresh_head` fixture at line ~97; `test_alembic_round_trip_is_clean` at lines ~363-364) work unchanged.

The new body MUST:

1. Validate `action` against `{"upgrade", "downgrade"}` at the top with `if action not in ("upgrade", "downgrade"): raise ValueError(action)` — preserves the existing input-validation behavior.

2. Build argv as a list literal: `cmd = ["uv", "run", "alembic", action, target]`. Match the justfile recipe exactly — `uv run alembic ...` not `python -m alembic ...` or bare `alembic ...` (the `uv run` prefix ensures the project venv is active, matching how CI and operators run migrations).

3. Resolve `cwd` to the repo root: `cwd = Path(__file__).resolve().parents[2]`. The existing helper already computes this path for `alembic.ini`; the subprocess uses the same root so `alembic.ini` is auto-discovered (no `-c` flag needed — the justfile recipe doesn't pass one either).

4. Wrap the synchronous `subprocess.run` in `asyncio.to_thread(...)` so the event loop is not blocked during the multi-second migration. Pattern:
```
import asyncio
import subprocess

result = await asyncio.to_thread(
    subprocess.run,
    cmd,
    cwd=str(cwd),
    capture_output=True,
    text=True,
    timeout=120,
)
```

5. Manually check `result.returncode` instead of passing `check=True` — this lets the error surface include `result.stdout` and `result.stderr` in the assertion failure for debugging. If the migration fails, the test reader will see the actual alembic error text instead of a bare `CalledProcessError`. Pattern:
```
if result.returncode != 0:
    raise AssertionError(
        f"alembic {action} {target} failed (exit {result.returncode}):\n"
        f"--- stdout ---\n{result.stdout}\n"
        f"--- stderr ---\n{result.stderr}"
    )
```
Do NOT use `subprocess.CalledProcessError` because its default `__str__` truncates output; a custom `AssertionError` with full output is dramatically more useful in pytest reports.

   - Add `import asyncio` to the existing top-level imports (the file already uses `asyncio.run` indirectly via pytest-asyncio so this is consistent).
   - REMOVE the lazy `from alembic import command` + `from alembic.config import Config` imports — they are no longer used. Run `grep -c "from alembic" tests/integration/test_migrate_0009.py` and confirm 0 hits after the edit.
